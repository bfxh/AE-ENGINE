## 2. 实时全局光照 (GI) 现状

### 2.1 DDGI — Dynamic Diffuse Global Illumination
**论文**: Majercik et al., "Fast Gradient-Domain Multiplication for Integral Illumination", I3D 2019
**作者**: Zander Majercik (McGill) / Jean-Philippe Guertin / Derek Nowrouzezahrai

- 3D 网格 light probe，每个 probe 存 8×8 irradiance (RGB) + depth
- 每帧 ray-march 部分更新（轮流），双缓冲
- 渲染时三线性插值 8 个邻居 probe
- 优势：纯 diffuse，无噪声，无需 denoiser
- 劣势：仅 diffuse，无法处理 caustics / glossy GI

**✅ 已完成** (2026-06-28): `nova_render/src/gi/ddgi.rs` 42276 字节，0 errors，8 单元测试通过。`n- DdgiUniform(224B) + ProbeData(16B) + DdgiScene trait`n- Ddgi struct（4 公共字段 + 12 私有 GPU 资源 Option）`n- DDGI_UPDATE_SHADER (compute 8x8): 球面采样场景颜色 → irradiance 3D storage texture`n- DDGI_RENDER_SHADER (fragment): 三线性插值 8 邻居 probe irradiance`n- 简化点：无 ray-march（仅单次投影采样）、depth 不参与遮挡、4 probes/frame round-robin

### 2.2 ReSTIR — Reservoir-based Spatiotemporal Importance Resampling
**论文**: Bitterli et al., "Spatiotemporal reservoir resampling for real-time ray tracing with dynamic direct lighting", SIGGRAPH 2020
**作者**: Benedikt Bitterli (NVIDIA) / Chris Wyman 等

**演进系列**:
- **ReSTIR DI** (2020) — 直接光照，千灯 60fps
- **ReSTIR GI** (2022, Ouyang) — 间接光照，1spp 路径复用
- **ReSTIR PT** (2022, Lin) — 全路径追踪，3090 上 40-80ms
- **ReSTIR PT Enhanced** (Lin et al., ACM TOG 2026) — 2-3× faster, FLIP 误差减半
- **ReSTIR BDPT** (Hedstrom et al., ACM TOG 2025) — 处理 caustics, MAPE 0.312 vs 1.368

**核心思想**: 加权蓄水池采样 (WRS) + 时空邻居复用，让每个像素"借用"邻居的好样本，等效采样数 O(M²) 增长而无需额外 ray

**开源参考**: 
- **Solari** (Bevy 0.17 官方插件) — `git clone https://github.com/bevyengine/bevy`，`cargo run --example solari --features bevy_solari`
- Pipeline: GBuffer Raster → ReSTIR DI → ReSTIR GI → Light Tile Presampling → World Cache → DLSS RR

### 2.3 Lumen (UE5)
**论文**: Wright et al., "Lumen: Real-time Global Illumination with Mega Lights in UE5", SIGGRAPH 2022

- 屏幕空间 probe + 远场 irradiance cache + 体素 cone trace 混合
- 不需要 RT 硬件，但开启 RT 后质量更高
- 50ms 预算内完成 1-bounce GI

### 2.4 NDGI — Neural Dynamic GI ⭐ 新
**论文**: CVPR 2026 (中国科学技术大学 + 浙江大学)
- 神经压缩框架 + 时序 lightmap
- 2D 三平面特征图（空间高频）+ 低分辨率 3D 特征图（时间低频）
- BC7-Simulation 训练策略（神经网络训练时模拟硬件 BC7 压缩）
- 虚拟纹理 (VT) 集成，按需 tile 解码，毫秒级
- 适用：预计算动态光照（昼夜交替），存储降低 10×

### 2.5 SSGI — Screen Space Global Illumination
- 类 SSAO，但采样邻居像素颜色而非深度
- 无法处理屏幕外几何，需要 RT 或 DDGI 补充

## 3. 实时光追 (RT) 现状

### 3.1 wgpu 24 已上游 RT 支持
- `wgpu::Features::EXPERIMENTAL_RAY_TRACING_ACCELERATION_STRUCTURE`
- Bevy 0.17 Solari 已验证可用
- nova_render 可直接对接（升级 wgpu features）

### 3.2 RT 应用清单
| 应用 | 算法 | 性能预算 |
|------|------|---------|
| RT Shadow | 1 spp shadow ray + denoise | 2-4ms |
| RT Reflection | 1 spp refl ray + denoise | 4-8ms |
| RT AO | 1 spp ambient ray + denoise | 2-4ms |
| RT GI | ReSTIR GI 1 spp + denoise | 8-15ms |
| RT Path Tracing | ReSTIR PT + DLSS-RR | 30-70ms |

### 3.3 Denoiser 选型
- **SVGF** (Schied 2017) — 开源，spatiotemporal variance-guided filter，无需 AI
- **NRD** (NVIDIA 2020) — 公开，REBLUR/SIGMA，工业级
- **DLSS-RR** (NVIDIA 2025) — 闭源，需 SDK，质量最高

## 8. nova_render 设计决策建议

1. **G-Buffer 阶段** → 升级为 **Visibility Buffer**（更省带宽，与 Nanite/cluster 兼容）
2. **GI 策略**:
   - 短期：DDGI（diffuse）+ SSGI（屏幕内）混合
   - 中期：wgpu RT 启用后 → ReSTIR DI + ReSTIR GI
   - 长期：ReSTIR PT + DLSS-RR
3. **后处理顺序**（借鉴 kajiya）:
   - TAA → MotionBlur → Exposure → Tonemap(ACES) → Bloom → CAS → DPI Upscale
4. **3DGS 集成**: 单独 splat pass，作为 G-Buffer 的"补充层"而非替换
5. **NDGI 时序 lightmap**: 用于静态大场景的昼夜变化（预计算 + 神经压缩）

## 11. Bevy 0.17 Solari Frame Breakdown (2025-09-20)

**来源**: JMS55 博客 "Realtime Raytracing in Bevy 0.17 (Solari)"
**仓库**: `git clone https://github.com/bevyengine/bevy && cd bevy && git checkout release-0.17.0`
**运行**: `cargo run --release --example solari --features bevy_solari,https`

### 11.1 Solari 帧管线（6 阶段）

| 阶段 | 算法 | 输入 | 输出 |
|------|------|------|------|
| 1. GBuffer Raster | 硬件光栅化 | 场景几何 | GBuffer (albedo/normal/depth/mesh_id) |
| 2. ReSTIR DI | Bitterli 2020 | GBuffer + 光源 | 直接光照 reservoir |
| 3. ReSTIR GI | Ouyang 2022 | GBuffer + 历史帧 | 间接光照 reservoir |
| 4. Light Tile Presampling | tiled light | 光源列表 | tile 内预采样 reservoir |
| 5. World Cache | radiance cache | ray hit | 时序复用 cache |
| 6. DLSS Ray Reconstruction | NVIDIA DLSS-RR | noisy color | 最终降噪图像 |

### 11.2 wgpu RT 启用方法（P2 准备）

```rust
// wgpu 24 RT 已上游化（不再需要 fork）
let (device, queue) = adapter.request_device(&wgpu::DeviceDescriptor {
    required_features: wgpu::Features::EXPERIMENTAL_RAY_TRACING_ACCELERATION_STRUCTURE
        | wgpu::Features::EXPERIMENTAL_RAY_QUERY
        | wgpu::Features::EXPERIMENTAL_RAY_HIT_VERTEX_RETURN,
    required_limits: wgpu::Limits::default(),
    memory_hints: wgpu::MemoryHints::default(),
}, None).await?;

// 创建 BLAS (Bottom Level Acceleration Structure)
let blas = device.create_blas(&wgpu::CreateBlasDescriptor { label: None, mode: BlasBuildMode::Build });
// 创建 TLAS (Top Level Acceleration Structure)
let tlas = device.create_tlas(&wgpu::CreateTlasDescriptor { label: None, mode: TlasBuildMode::Build });
```

### 11.3 nova_render P2 落地路线

1. **启用 wgpu RT features** — 在 Cargo.toml + device 创建时启用 RT features
2. **BLAS/TLAS 管理** — 创建 src/rt/acceleration_structure.rs
3. **ReSTIR DI** — 创建 src/gi/restir_di.rs（Bitterli 2020，reservoir-based importance sampling）
4. **ReSTIR GI** — 创建 src/gi/restir_gi.rs（Ouyang 2022，1spp 路径复用）
5. **DLSS-RR 集成** — 对接 dlss_wgpu crate（NVIDIA 闭源 SDK）
6. **World Cache** — 创建 src/gi/world_cache.rs（radiance cache 时序复用）

### 11.4 关键限制（Solari 2025 现状）

- 仅支持 Vulkan + DX12（Metal RT 待补全）

## 12. NVIDIA Zorah 2025 技术拆解

**来源**: NVIDIA GDC 2025 演讲 "Path Tracing Nanite in NVIDIA Zorah" (gdc25-gdc1002) + RTX Kit 官方文档 + agraphicsguynotes 博客 "Understanding The Math Behind ReSTIR GI" (2025-05-09) + Adobe Substance 3D 团队访谈 (2025-04-03)

**背景**: Zorah 是 NVIDIA 为 Blackwell / GeForce RTX 50 Series 发布的旗舰技术 demo，2025 CES 首发，GDC 2025 公开源码分支（基于 UE5 NvRTX 分支）。源码下载: `https://dlss.download.nvidia.com/demos/zorah/ZorahSample_UE5_Source_1.1.0.7z`

### 12.1 场景规模与技术目标

| 指标 | 数值 |
|------|------|
| 几何总量 | 800M+ triangles（数百万元素级） |
| 动态光源 | 3500+ lights（artist 持续调整） |
| 材质复杂度 | UE5 Substrate 多 Slab 堆叠（tarnished silver / patterned gemstones / slubbed silk 等） |
| 目标硬件 | RTX 5090 旗舰，向下兼容 RTX 20+ |
| 帧率目标 | 4K 240 FPS（依赖 DLSS 4 Multi Frame Generation） |

### 12.2 ReSTIR 配置（核心 GI 算法）

Zorah 的 light transport 由 Daqi Lin（NVIDIA Research）+ Jiayin Lin 实现，集成在 NvRTX 分支的 UE5 中：

| 变体 | 用途 | 论文 |
|------|------|------|
| **ReSTIR DI** | 直接光照（千灯场景）approximation | Bitterli 2020 |
| **ReSTIR GI** | 间接 diffuse GI（1-bounce 路径复用） | Ouyang 2022 |
| **ReSTIR PT** | 全路径追踪（mirror reflection + multi-bounce GI） | Lin 2022 — Zorah 中由 Daqi Lin 实装 |
| **ReSTIR BT** | ReSTIR GI 的高级变体（Bias-corrected Target） | 用于 GI signal approximation |

**关键事实**（来自 agraphicsguynotes）: Zorah 中所有 light bounce 都是**实时动态评估**的，无任何 baked GI。ReSTIR GI 单独运行即可产生与 ReSTIR PT 相当的 diffuse GI 信号；ReSTIR PT 的主要改进是 mirror reflection。

### 12.3 RTX Mega Geometry (CLAS) 用于 Nanite ray tracing

Zorah 是首个将 Nanite 几何与完整 path tracing 结合的生产级 demo：

- **CLAS (Cluster Acceleration Structure)** — OptiX 9 / DXR / Vulkan (`VK_NV_cluster_acceleration_structure`) 引入的新 AS 数据结构
- 输入: 用户定义的 256-triangle cluster
- 输出: Cluster GAS（类似传统 GAS，但可 GPU-driven build）
- 解决问题: Nanite 几何每帧 triangle 数/connectivity 都变化，传统 BVH refit 失效
- 性能: Alan Wake 2 实测 -1GB VRAM, +13% FPS on RTX 4090
- RTX 50 (Blackwell) 第四代 RT Cores 新增:
  - **Triangle cluster intersection engine** — 2× ray-triangle intersection 吞吐 vs 3rd gen
  - **Triangle cluster compression engine** — 压缩 cluster 数据省显存

UE5 集成: CVAR `r.Nanite.RayTracing=1`（read-only，需在 default engine ini 设置），MegaGeometry 自动为 Nanite 构建 AS。

### 12.4 Cooperative Vectors + Neural Shaders

Zorah 集成了 RTX Neural Materials（RTX Kit 组件）：

- **Cooperative Vectors** (OptiX 9 / DX12 Agility SDK 1.717 / Vulkan `VK_NV_cooperative_vector`) — 让 shader 单线程内调用 Tensor Cores 做矩阵-向量乘
- **RTX Neural Materials** — 将复杂多层 BSDF（如 slubbed silk 多 anisotropic lobe）压缩为 2x64 + 3x32 MLP，实时推理
- **RTX Neural Texture Compression (NTC)** — 8× 纹理内存压缩，与 BC7 等效质量
- **RTX Neural Faces** — 生成式 AI 实时渲染 photorealistic 面孔（仅 raster + 3D pose 输入）
- Slang (shader-slang) + SlangPy 用于训练 MLP，部署用 Cooperative Vectors

### 12.5 DLSS 4 Multi Frame Generation

Zorah 在 RTX 5090 上 4K 完全 path-traced 240 FPS，依赖 DLSS 4：

| 组件 | 改进 |
|------|------|
| **Multi Frame Generation** | 每传统帧生成 3 个中间帧（4× total），仅 Blackwell 支持 |
| **Frame Generation AI model** | 40% 更快，30% 更省 VRAM，每帧只跑一次 |
| **Optical Flow** | 用 AI 模型替代硬件 OFA，效率更高 |
| **Flip Metering** | Display controller 内置 frame pacing，降低 micro-stutter |
| **Super Resolution (transformer)** | 图形行业首个实时 transformer 应用，2× 参数量 4× 计算量 |
| **Ray Reconstruction (transformer)** | 同上，path tracing denoising 质量提升 |
| **NVIDIA Reflex 2 + Frame Warp** | 渲染前根据最新 mouse input 更新，延迟降 75% |

### 12.6 nova_render 落地建议

1. **ReSTIR PT 是终极目标但非 P0** — Zorah 路径需 DLSS-RR + RTX 50，nova_render 短期仍走 DDGI + SSGI 路线
2. **CLAS 对接 cluster_lod.rs** — nova_render 已有 Nanite 风格 cluster LOD，可在 P2 启用 wgpu RT 后对接 CLAS（Vulkan `VK_NV_cluster_acceleration_structure` 扩展）
3. **Neural Materials 探索** — Cooperative Vectors 在 wgpu 中尚无原生支持（详见 §16），可作为 P3 实验性特性
4. **DLSS 4 不可替代** — 闭源 SDK，nova_render 短期走 TAA + CAS，长期可考虑 OpenImageDenoise / NRD 开源路径

- 模块化管线: Preprocess (cull + indirect draw setup) → RadixSorter (GPU depth sort) → Renderer
- 显示模式: splat / ellipse / point
- 支持 PLY 和 **SPZ** 格式（Niantic 压缩格式，详见原文档 §4.1）
- Multi-model 渲染（`multi-model` feature）
- Selection 工具（rectangle / brush，`selection` feature）
- 配套 `wgpu-3dgs-editor` crate（0.7.0）— selection + modifier (WGSL/WESL 自定义 shader)
- WGSL shaders 用 WESL 打包，可替换

### 13.3 3DGUT (NVIDIA, nv-tlabs/3dgrut)

**仓库**: `https://github.com/nv-tlabs/3dgrut`
**论文**: Wu et al., "3DGUT: Enabling Distorted Cameras and Secondary Rays in Gaussian Splatting", CVPR 2025 (Oral)
**arXiv**: `https://arxiv.org/abs/2412.12507`
**License**: 开源（NVIDIA Research）

**核心创新**: 用 Unscented Transform 替代 3DGS 中的 EWA splatting
- 支持非线性相机投影（fisheye / rolling shutter）— 自动驾驶 / 机器人传感器仿真
- 与 3DGRT（3D Gaussian Ray Tracing）协同，支持 secondary rays（反射 / 折射）
- 三种渲染模式:
  - **3DGRT** — 纯 ray tracing，支持 secondary rays，最慢但最准
  - **3DGUT** — 纯 rasterization + UT，最快，支持 distorted camera
  - **3DGRUT** — 混合：primary ray rasterization + secondary ray tracing
- 集成到 gsplat 库（Apache 2.0），与 Nerfstudio 兼容
- **依赖 CUDA + RT cores**，不支持 wgpu

## 14. 神经 GI 最新进展（2025-2026）

### 14.1 NDGI — Neural Dynamic GI (CVPR 2026)

**论文**: Wu, Zhou, Zhou, Huang (USTC) + Li (Zhejiang Univ.), "Neural Dynamic GI: Random-Access Neural Compression for Temporal Lightmaps in Dynamic Lighting Environments", CVPR 2026
**PDF**: `https://openaccess.thecvf.com/content/CVPR2026/papers/Wu_Neural_Dynamic_GI_*.pdf`
**主页**: `https://magicdawnlab.github.io/`

**核心思想**: 时序 lightmap 的神经压缩
- **问题**: 动态光照（昼夜交替）需预计算多套 lightmap，存储爆炸
- **方案**: 时空解耦的混合特征图
  - 2D 三平面特征图（uv）— 空间高频（128×128，4 channels）
  - 低分辨率 3D 特征图（ut）— 时间低频（64×24）
  - 轻量 MLP 解码器: (x, y, t) → radiance
- **BC-Simulation 训练策略**: 训练时模拟 BC7 硬件压缩损失，最终特征图可直接 BC7 压缩
- **Virtual Texturing 集成**: tile 化按需推理，一次推理多帧复用，毫秒级
- **效果**: 相比 PRT 和 NTC，更高压缩比 + 更好光照质量 + 更低运行时开销
- **数据集**: 将公开预计算的时序 lightmap 数据集

**适用**: 大场景动态昼夜 GI（如 wasteland 项目的大地图）

### 14.2 TransGI — 跨场景神经 GI (arXiv 2025-06)

**论文**: Deng (Tsinghua) + Han (Huawei) + Fang (Tsinghua/BNRist), "TransGI: Real-Time Dynamic Global Illumination With Object-Centric Neural Transfer Model", arXiv:2506.09909 (2025-06-11)

**核心思想**: object-centric neural transfer model 替代 BSDF
- **问题**: 传统 BSDF 需要千次 ray 评估才收敛；实时方法只支持 diffuse 牺牲质量
- **方案**: 
  - MLP decoder + vertex-attached latent features（每顶点 1D 向量）
  - 网络 <1MB，per-vertex latent 极小
  - 输入: vertex latent + outgoing direction + G-Buffer
  - 输出: transfer coefficients（支持 glossy）
- **Light Probe 系统**:
  - SH light probes 捕获场景 radiance
  - **Across-probe radiance sharing** — 跨 probe 共享降低生成成本
  - Probe 间三线性插值
- **实现**: compute shaders (DX12) + CUDA 神经网络，**<10ms/frame**
- **优势**: 支持动态光照 + glossy GI，存储 <1MB（vs 传统 PRT 数十 MB）

### 14.3 NIRC — Neural Incident Radiance Cache (EG 2025)

**论文**: Dereviannykh, Klepikov, Hanika, Dachsbacher (KIT), "Neural Two-Level Monte Carlo Real-Time Rendering", Computer Graphics Forum 44(2), Eurographics 2025
**arXiv**: `https://arxiv.org/html/2412.04634`
**DOI**: 10.1111/cgf.70050

**核心思想**: Two-Level Monte Carlo (MLMC) + Incident Radiance Cache
- **vs NRC (Müller 2021)**: 
  - NRC 缓存 outgoing radiance（需 ray cast 后查询）
  - NIRC 缓存 **incident radiance**（避免 ray cast before lookup）
- **Two-Level MC 估计**: 
  - L1 = radiance cache integral（高 spp，用 NIRC 近似）
  - L2 = residual error integral（低 spp path tracing 补偿 bias）
  - 总估计 = L1 + L2，理论上无偏
- **NIRC 评估速度**: 比 path tracing sample 快 **2-25×**
- **架构**: fully-fused tiny neural network + multiresolution hash encoding
- **在线训练**: 动态场景适应性强，可与 SVGF / NRD 等降噪器叠加
- **测试**: Bistro scene RTX 3080 1080p, 1 spp

**关联工作**: SIGGRAPH Asia 2025 Technical Communications 有 "Neural Radiance Cache Implementation on Mobile GPU"（Samsung + Traverse Research）— 验证 NRC 在移动端的可行性（compute shader 实现，~31ms full frame）。

### 14.4 nova_render 可落地方案

| 方案 | 实现复杂度 | 硬件要求 | nova_render 适用阶段 |
|------|-----------|---------|---------------------|
| **NDGI 时序 lightmap** | 中（MLP + VT） | 低（无需 RT） | ✅ P2 — 大场景昼夜动态光 |
| **TransGI transfer model** | 高（MLP + probe 系统） | 中（需 compute shader MLP） | ⚠️ P3 — 替代 PRT |
| **NIRC + MLMC** | 极高（在线训练 + tiny MLP） | 高（需 RT） | ❌ P3+ — 需先完成 wgpu RT 启用 |

**推荐路径**: 
1. **P2 优先 NDGI** — 与 nova_render 已有 `gi/ddgi.rs` 互补，NDGI 处理静态大场景昼夜，DDGI 处理动态局部 GI
2. NDGI 的 BC-Simulation 思路也适用于现有 lightmap 系统
3. NIRC 需等 wgpu RT 启用（P2 完成）后再评估

## 15. GPU-Driven Pipeline 2025

### 15.1 RTX Mega Geometry (CLAS) 详解

**文档**: 
- NVIDIA OptiX 9 blog (2025-04-24): `https://developer.nvidia.com/blog/fast-ray-tracing-of-dynamic-scenes-using-nvidia-optix-9-and-nvidia-rtx-mega-geometry/`
- Vulkan samples (2025-02-06): `https://developer.nvidia.com/blog/nvidia-rtx-mega-geometry-now-available-with-new-vulkan-samples/`
- 开源: `https://github.com/NVIDIA-RTX/RTXMG`
- Vulkan 扩展: `VK_NV_cluster_acceleration_structure`, `VK_NV_partitioned_acceleration_structure`

**CLAS (Cluster Acceleration Structure) 设计**:
- 输入: 用户定义的 triangle cluster（最多 256 triangles）
- 输出: Cluster GAS（顶层 GAS 的输入）
- **核心优势**: 
  - 提供 refit 的速度 + full rebuild 的灵活性和 trace 性能
  - Cluster 模式可复用（相同 topology 的 mesh 共享 CLAS 模板）
  - GPU-driven build（CPU 不参与 BVH 管理）
- **Cluster Templates** (OptiX API): 预计算 CLAS build 大部分工作，加速重建 1 个数量级

**Vulkan 4 个官方 sample**:
1. **vk_animated_clusters** — 每帧三角形变形的集群 ray tracing
2. **vk_partitioned_tlas** — 部分场景变化时只重建 TLAS 子集（100K physics objects）
3. **vk_tessellated_clusters** — 动态 tessellation + displacement 的 path tracing
4. **vk_lod_clusters** — 连续 LoD + RAM→VRAM streaming

**配套库**: `nv_cluster_builder`, `nv_lod_cluster_builder` — 几何聚类工具

### 15.2 Mesh Shaders 在 wgpu 24+ 支持

**wgpu feature**: `Features::EXPERIMENTAL_MESH_SHADER`（需同时启用 `EXPERIMENTAL_MESH_SHADER`）
**支持平台**: Vulkan + Metal（DX12 路径在规划中）
**文档**: `https://wgpu.rs/doc/wgpu/struct.Features.html`

**Mesh Shader 模型**（来自 Evergine 2025-10 文档）:
- 经典管线: 3 fixed-function + 5 programmable stages
- Mesh Shader 管线: 2 fixed-function + 3 programmable stages
- 新增 stage:
  - **Task Shader** (DX12: Amplification Shader, Metal: Object Function)
  - **Mesh Shader** (替代 Vertex + Geometry + Hull)
- **Meshlet 标准尺寸**: 64 vertices + 126 triangles（GPU 最优）
- **Cluster 桥接**: VK_NV_cluster_acceleration_structure + VK_EXT_mesh_shader 协同工作
- 工具: MeshOptimizer (Arseny Kapoulkine) — 行业标准 meshlet 生成库

**WebGPU 路线图**（kaelan.fyi 2025-01）:
- Bindless resources 是前置依赖（mesh shader + RT 都需要）
- 优先级: Bindless → Subgroups → Mesh Shaders → Ray Tracing
- WebGPU 2.1 (2026) 可能引入 mesh shaders

### 15.3 DirectStorage 纹理流送

**API**: Microsoft DirectStorage 1.3 (2025)
**文档**: `https://learn.microsoft.com/en-us/gaming/gdk/docs/features/console/storage/directstorage/directstorage-overview`

**核心改进**:
- 旧路径: storage → system RAM → CPU decompress → RAM → GPU VRAM（多次拷贝，CPU bound）
- DirectStorage: storage → mapped buffers → **GPU decompress (GDeflate)** → VRAM
- 目标: 50K IOPS @ <10% 单核 CPU
- **GDeflate**: 64 KiB tile 粒度的并行解压格式，GPU compute queue 执行
- 队列: 1 compute queue + 2 copy queue（compressed in / decompressed out）
- 兼容: BCPACK + DEFLATE + swizzle 可叠加

**实测**:
- Intel Arc A770 demo: ~2.7× asset loading 加速 vs CPU decompression
- Alan Wake 2 (RTX Mega Geometry + DirectStorage): -1GB VRAM, +13% FPS
- 实际游戏（Forspoken / Ratchet & Clank: Rift Apart / Horizon Forbidden West）: 通常 1-2 秒加载改善，未达"无加载"承诺

**局限**:
- Windows 11 才有完整 GPU decompression（Win10 仅 I/O 优化）
- 需要 NVMe PCIe Gen3+ （Gen4 性价比最优）
- 开发者采用缓慢，多数游戏未集成

### 15.4 nova_render cluster_lod.rs 对接 CLAS 路线

**当前状态**: `nova_render/src/renderer/cluster_lod.rs` 已实现 Nanite 风格 cluster LOD（详见 §17）

**对接 CLAS 的步骤**（P2 阶段）:

1. **wgpu 启用 RT + Mesh Shader**:
   ```rust
   required_features: wgpu::Features::EXPERIMENTAL_RAY_TRACING_ACCELERATION_STRUCTURE
       | wgpu::Features::EXPERIMENTAL_RAY_QUERY
       | wgpu::Features::EXPERIMENTAL_MESH_SHADER,
   ```

2. **cluster_lod.rs 输出 cluster 数据**: 当前生成 meshlet（64 vert / 126 tri），CLAS 需 256-triangle cluster — 调整 cluster 大小或分组

3. **BLAS → CLAS 转换**: 创建 `nova_render/src/rt/clas.rs`，将 cluster_lod 输出的 cluster 喂给 Vulkan `VK_NV_cluster_acceleration_structure`

4. **TLAS partitioned rebuild**: 场景部分变化时只重建子集（对齐 `VK_NV_partitioned_acceleration_structure`）

5. **Mesh Shader 渲染路径**: cluster_lod.rs 可同时驱动 mesh shader rasterization + CLAS ray tracing，几何数据共享

**降级方案**: 非 NVIDIA GPU（无 VK_NV 扩展）回退到现有 mesh shader 路径 + 传统 BLAS

## 16. DLSS 4 + Cooperative Vectors + Neural Shaders

### 16.1 DLSS 4 技术拆解

**发布**: CES 2025 (2025-01-06), Blackwell / RTX 50 Series
**支持游戏**: 175+ (截至 2025-08 Gamescom)
**UE5 集成**: DLSS 4 plugin 支持 UE 5.2 - 5.6
**Streamline SDK**: 自定义引擎集成路径

**5 大组件**:

| 组件 | 功能 | 硬件要求 |
|------|------|---------|
| **Multi Frame Generation (MFG)** | 每渲染帧生成最多 3 个中间帧（4× 总帧率） | RTX 50 only (5th gen Tensor Cores) |
| **Super Resolution (transformer)** | 低分辨率 → 高分辨率，2× 参数 4× 计算 vs DLSS 2 CNN | RTX 20+ (beta) |
| **Ray Reconstruction (transformer)** | path tracing denoising，transformer 架构 | RTX 20+ (beta) |
| **DLAA (transformer)** | 抗锯齿，原生分辨率 | RTX 20+ (beta) |
| **Reflex 2 + Frame Warp** | 渲染前根据 mouse input 更新，延迟 -75% | RTX 50 最优 |

**MFG 关键技术**:
- 新 frame generation AI model: 40% faster, 30% less VRAM, 每帧只跑一次
- AI optical flow 替代硬件 OFA
- **Flip Metering**: display controller 内置 frame pacing，CPU 不参与
- 必须搭配 NVIDIA Reflex 维持输入响应

**性能数据**: Cyberpunk 2077 4K RT, RTX 5090, DLSS 4 MFG vs 传统渲染: **8× FPS**, 延迟减半, 画质提升

### 16.2 Cooperative Vectors (Vulkan / DX12 / OptiX)

**Vulkan 扩展**: `VK_NV_cooperative_vector`（Vulkan SDK 1.4+）
**DX12**: Agility SDK 1.717.0-preview, Shader Model 6.9 preview
**OptiX**: OptiX 9.0
**Slang**: 2025-01-30 起支持

**解决的问题**: shader 单线程做 MLP 推理时如何用 Tensor Cores
- CUDA SIMT 模型需要完整 warp 才能用 Tensor Cores
- Ray tracing 编程模型独立处理线程，无法保证 warp 完整
- Tensor Cores 提供矩阵-矩阵乘，但 ray tracing 每线程只需向量-矩阵乘

**Cooperative Vectors API**:
- 允许部分 inactive thread 的 warp 做矩阵运算
- 跨架构向前向后兼容
- 单线程指定向量数据，硬件重映射为矩阵-矩阵运算利用 Tensor Cores
- 关键函数: `coopVecMatMulAddNV(result, input, matrix, bias, M, K, ...)`

**最佳性能条件**:
- Warp 内所有线程评估同一 MLP（用 Shader Execution Reordering SER 保证）
- Warp 内 MLP 权重相同

**VS Cooperative Matrix (VK_KHR_cooperative_matrix)**:
- Cooperative Matrix: subgroup 共享，需 uniform control flow
- Cooperative Vector: 每线程独立矩阵，SIMT 友好，更适合 ray tracing
