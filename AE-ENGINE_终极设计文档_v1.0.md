# AE-ENGINE (AE-ENGINE) 终极设计文档

> **版本**: v1.0
> **日期**: 2026-06-05
> **核心原则**: 还原现实 — 一切行为准则按真实物理、化学、生物规则运行

---

## 一、游戏概述

### 核心定义

一款由底层科学规则驱动的单机物理沙盒生存制造游戏。玩家扮演从培养舱中苏醒的克隆体，在黏菌覆盖的废土上，用双手拼装一切——武器、工具、建筑、甚至生命本身。没有固定配方，没有预设动画，所有玩法由物理、化学、生物规则的实时交互涌现产生。世界会记住玩家的每一个选择，并自行演化。

### 核心哲学

- **规则取代配方**：功能推导引擎分析几何结构与材料属性，实时赋予物品功能标签和数值，而非查表。
- **万物交互统一**：物理、化学、生物不再区分为独立系统，而是统一为元体属性响应。
- **真实不妥协**：所有规则在后台持续运行，不因玩家距离或视野而裁剪精度，只区分计算与呈现。
- **废土自有生命**：世界在玩家离线时继续演化，玩家行为留下永久痕迹，NPC拥有长期记忆和自主目标。

## 二、世界观与叙事

### 1. 时间线

- **繁荣纪元**：人类共和体的黄金时代，科技巅峰持续约2000年。国家、民族、宗教消亡，AI、聚变能源、纳米制造和基因编辑普及。
- **繁荣纪元末期**：约200年间，内部裂痕渐显——AI自治运动、基因分化运动、星际殖民地离心倾向、纳米安全争议。
- **漫长战争**：持续约100年。来自太阳系外的黏菌孢子云从天而降，直接覆盖陆地。黏菌是一种无意识的环境改造程序，分解碳基有机物并自我复制，人类无法根除。战后重建委员会分裂为学院、军团、企业。
- **废土时代**：战后至今约600年。玩家在第600年从烬城废墟边缘的培养舱意外苏醒，作为"伊甸重启计划"第47批次克隆体，记忆空白。

### 2. 核心冲突

黏菌并非敌人，它是一个已灭绝文明的墓碑——一个永不完成的殖民程序。真正的冲突是幸存者之间对文明遗产、生存方式和权力结构的争斗。

### 3. 主要势力

- **学院**：战前超然派科学家后裔。拥有完整战前科技数据库，对废土采取"观察但不干预"策略，视一切为实验样本。内部存在不干预派、有限干预派和直接干预派。
- **军团**：委员会军事派后裔，拥有废土最大武装力量。以"秩序高于自由"为信条，试图武力统一废土。元帅与玩家同批次的克隆体，但被学院秘密控制认知滤网。
- **企业**：委员会工业派后裔，由欧若拉（能源）、克雷莫（武器）、梅迪奇（生物医药）、瓦伦（物流）和新人会（黑市）五大家族组成的商业联盟。通过信用货币和贸易网络控制废土经济。
- **其他势力**：守夜人（崇拜黏菌的苦行者）、合成体（战前AI遗留的机器生命）、深海族（海底基因改造后裔）、轨道居民（近地轨道空间站幸存者）、绿洲联邦（未被黏菌污染的飞地）、菌丝议会（黏菌集体意识节点）。

### 4. 可选种族与开局

- **废土客**：战后未进入避难所的幸存者后裔。自然选择赋予部分黏菌抗性，拥有部落网络和丰富的环境知识。开局装备简陋但生存经验丰富。
- **虫族**：来自外星系的观察者，拥有基因重组能力。可以建造母巢，培育生物武器，对人类基因多样性抱有学术兴趣。开局需建立巢穴，与废土居民关系微妙。
- **克隆人**：伊甸重启计划的产物，记忆空白但身体经过基因优化。植入认知滤网，可主动屏蔽黏菌干扰、增强学习能力，并与旧世界设备进行神经接口连接。

### 5. 新手村机制

玩家在烬城废墟旁的废弃避难所（新手村）苏醒，资源有限。当玩家准备好离开时，打开出口大门，踏入废土。离开一定距离后，第一个Boss（如母巢追猎者或军团巡逻机甲）会摧毁或封锁入口，无法返回。废土没有难度选择，适应性生存是唯一的法则。

## 三、核心机制

### 1. 制造系统

- **零件拼装**：所有物品由原子几何体（石片、木棍、藤蔓等）通过Socket吸附和物理约束连接而成。
- **功能推导引擎**：实时分析组合体的几何特征（锐边、尖端、空腔、平面）、材料属性和质量分布，输出连续的功能置信度（如劈砍0.89、穿刺0.12）与实时属性（伤害、耐久、重心、攻速）。
- **蓝图记录**：任何有效组合会被自动记录为蓝图，可供后续引导复现，并可分享、交易。蓝图本质是结构参数和材料清单，不依赖预设配方。

### 2. 武器与护甲

- **统一判定**：所有物体都可攻击。武器的伤害由刃材硬度、有效质量、速度系数和刃口微观状态实时计算。
- **护盾系统**：繁荣纪元遗留的引力偏导护盾，通过驱动引力子产生时空曲率。高速物体（如子弹）能量-动量密度大，被轻易偏转；低速制导武器（如弩矢、手雷）和近战攻击可穿透。护盾能量有限，可被饱和攻击耗尽。
- **破坏分级**：物品和建筑基于真实材料力学，经历表面损伤、局部变形、部件断裂到完全毁坏四级破坏，所有改变永久保存。

### 3. 化学引擎

- **第一性原理推导**：不查表，基于电负性、键能、热力学数据（由RDKit离线生成并导入）实时计算反应产物、速率和热效应。反应按动力学缓慢推进，而非瞬间完成。
- **实验系统**：玩家可通过火焰颜色、沉淀反应、显微镜观察、基因测序等方式主动探索物质规律，发现新配方。知识自动记录于个人日志。

### 4. 生物引擎

- **基因Token系统**：每个Token对应真实生物学功能（如ACTN3、MSTN），决定性状和代谢。玩家可通过训练上调/下调表达，受损伤后修复强化，或用基因编辑器主动拼接新Token。
- **成长体系**：摒弃经验值与等级。用进废退（适应性表达）、超量恢复（损伤修复）和基因编辑三条路径协同作用，所有改变均伴随潜在副作用（如代谢失衡、神经适应性代价）。
- **人造生命**：后期可设计新物种，基因编辑→细胞核移植→人造子宫培养，全程需维持内环境稳态，产出物受基因Token规则严格约束。

### 5. NPC智能

- **分层心智**：战略层GOAP动态规划，战术层效用AI评分，执行层行为树。所有NPC与玩家共享同一套世界规则，能自主采集、制造、交易、战斗。
- **不降智原则**：通过时间切片交错更新、班组共享心智、视野外预测推演，使所有NPC在玩家感知中始终保持合理智能，不因距离被裁剪。
- **长期记忆**：NPC会记住玩家的行为（帮助、背叛、污染水源等），并影响后续态度、对话和任务生成。关系网络动态演化。

### 6. 实验系统

- **宏观**：火焰测试、硬度刻划、沉淀反应、毒性喂食。
- **介观**：光学显微镜观察细胞、微生物培养、晶体结构。
- **微观**：质谱分析、基因测序、X射线衍射——需修复繁荣纪元遗留设备。

## 四、世界规则层：从元体到结构场

### 1. 交互溶解架构

游戏底层不存在独立的物理引擎、化学引擎和生物引擎。所有实体统一为**元体**，每个元体携带一个完整的属性向量，包含：

- **基础物理量**：质量、密度、硬度、韧性、弹性模量、屈服强度、极限强度。
- **化学量**：元素组成、化学键类型、反应活性、酸碱度、氧化还原电位。
- **生物量**：基因Token序列、代谢速率、生长速率、修复速率、神经信号强度。

**交互机制**：当两个元体在空间上接近时，系统读取双方的属性向量，通过统一的响应函数计算输出：

- 力向量（改变运动状态）
- 属性变化量（修改双方或单方的属性值）
- 生成列表（可能产生新的元体，如气体、碎片）

响应函数基于元素周期表、化学键类型、代谢通路等基础规则自动匹配。所有交互结果在同一物理步长内生效，没有先后顺序。常见的交互结果被三级哈希索引缓存，加速复用。

### 2. 结构场

- **目的**：解决元体成对交互无法高效处理宏观层级耦合（如建筑承重、机械传动）的问题。
- **实现**：当建筑、机械等结构体被组装完成后，系统自动分析元体间的约束关系，生成约束图，并计算每个元体的结构场参数——包括其在层级中的深度、所属结构分组、是否为关键承重节点、应力传播的上下游邻居。
- **效果**：外力作用时，系统沿结构场预编译的路径快速传播应力，激活受影响的局部元体交互，避免遍历整个结构。大型坍塌被自然分阶段处理，计算量可控。

### 3. 统一数据层

所有元体属性、结构场参数、状态标志（冻结/激活/受损）都存储在统一ECS组件数组中。物理、化学、生物的交互直接读写同一份数据，无需跨系统通信。渲染层（Shader）根据ECS数据动态生成视觉效果，如氧化变色、肌肉形变。

### 4. 确定性

物理步进基于Rapier的确定性模式，并用定点数替换浮点数以保证跨平台一致性。交互响应函数和结构场传播路径在给定状态下均是纯函数，相同输入必然得到相同输出。

## 五、技术架构

### 1. 多语言分工与硬件目标

以Godot 4（深度改造版）为客户端框架，C++编写GDExtension桥接层，Rust编写核心确定性引擎。本地同机运行Go（全局场服务器）和Java（宏观战略服务器）子进程，Python用于离线资产管线。目标硬件：i5-7000 / RTX 4060 8G / 10GB RAM，1080p 60fps。

| 语言 | 职责 | 关键细节 |
|------|------|----------|
| Rust | 核心确定性引擎 | 元体数据结构、交互响应、结构场、物理步进、缓存系统、存档与事件溯源。通过GDExtension暴露给Godot。 |
| C | 底层数学与汇编封装 | 定点数运算、SIMD向量内核（AVX2/NEON）、三角函数近似、内存池分配器。 |
| 汇编 | 极致热点加速 | 仅用于批量碰撞检测和约束求解最内层循环，保留C语言fallback。 |
| C++ | Godot集成与编辑工具 | GDExtension桥接、渲染管线修改（SDFGI/FSR）、Tracy性能分析、L1内部开发工具（材料编辑器、规则调试器等）。 |
| Go | 全局场服务器（必选） | 风场、磁场、热扩散、声场等大规模低频计算（10-20Hz），通过Unix Socket与Rust核心通信。 |
| Java | 宏观战略服务器（必选） | 势力版图、经济、生态、遗传演化等分钟级~天级模拟，使用GraalVM Native Image编译为本地可执行文件，通过gRPC/REST通信。 |
| Python | 离线资产管线 | ComfyUI/Flux生图、TRELLIS/Hunyuan3D生成3D模型、Blender自动化精修、微型AI模型训练，完全不嵌入运行时。 |

### 2. 缓存与存储优化

- **交互缓存**：三级哈希索引（精确匹配→模糊分类→条件匹配）+ LRU-K淘汰 + 布隆过滤器预筛。
- **转存图**：所有交互历史以列式存储序列化到SSD，支持因果回溯、物品历史查询和开发者分析。采用日志结构顺序写入，后台定期合并快照。
- **SSD优化**：元体数据4KB页对齐，冷热数据分层存储（SLC/TLC/QLC），主动发送TRIM指令回收废弃块。

### 3. 资产管线

AI生成模型 → Open3D自动几何质检（非流形、破洞、面数） → MeshCNN/PointNet++语义分割 → Blender脚本精修（减面、重拓扑、UV展开、碰撞体生成） → 导出glTF → Godot导入时自动挂载功能组件。不合格资产自动回退至预制模型库。

## 六、开发计划与路线

1. **原型期**：搭建Godot+Rust GDExtension，实现元体成对交互（铁+酸腐蚀），验证功能推导引擎。
2. **核心期**：引入结构场约束图，处理简单建筑破坏；加入Go风场和Java生态模拟；实现三级缓存和转存图。
3. **扩展期**：完善化学/生物规则库（RDKit+BioModels），加入基因Token系统和母巢AI，开放L2模组工具。
4. **打磨期**：汇编SIMD热点加速，SSD深度优化，全平台自适应与帧率锁；完善UI/UX、单机特性、开源分享生态。

## 七、已知缺陷与应对

| 缺陷 | 缓解措施 |
|------|----------|
| 大型建筑坍塌时结构场传播路径可能过深 | 结构分组并行处理，单帧限制最大计算深度，剩余顺延至后续帧。 |
| 结构场首次生成大型约束图可能卡顿 | 后台线程异步生成，建造完成后有1~2秒延迟才可被破坏，玩家无感知。 |
| 连续介质效应（冲击波）需全局场模拟 | 由Go场服务器计算宏观等效力，精度满足游戏需求，避免离散粒子计算。 |
| 多语言调试困难 | 所有跨语言通信基于Protobuf，交互历史转存图统一记录跨系统因果链。 |
| 玩家可能发现最优解并广泛传播 | 不干预。最优解是玩家对规则深刻理解的奖励，开发者仅维护规则本身。 |
| 离线演化补算可能耗时 | 补算仅在读档时执行一次，使用事件流重放，并提供进度提示。 |

## 八、玩家体验与内容生态

### 1. UI/UX

- **沉浸式界面**：所有UI为世界内3D实体（全息控制台、投影日志），无弹出窗口。物品栏环绕角色身体，武器直接手持。
- **认知滤网接口**：血量、状态等以倾斜半透明元素显示，静止5秒后自动隐藏。玩家可调灵敏度，滤网可被干扰或入侵。
- **多设备自适应**：自动检测键盘/手柄/触屏并切换按键提示。

### 2. 开源创作与分享

- **蓝图系统**：所有玩家创造物（武器、基因序列、化学反应式）可自动生成蓝图，并以MIT、CC BY等开源协议通过Steam创意工坊或本地文件分享。
- **社区知识库**：游戏内可浏览、验证其他玩家发现的配方和物种，标记为"已验证"或"存疑"。
- **模组支持**：提供模组开发工具包（C++/Python），支持新零件、新材料、新AI行为。模组在沙盒中运行，冲突自动检测。

### 3. 单机独有特性

- **世界离线演化**：存档时记录时间戳，读档时根据离线时长加速补算宏观变化（势力兴衰、生态演变、建筑老化），世界永远在流动。
- **永久痕迹**：玩家建造的基地、污染的水源、杀死的Boss都会留下持久影响，无存档重置。
- **死亡与遗产**：硬核模式下永久死亡，存档转为"观察者模式"；或可寻找自己的尸体回收装备。
- **叙事涌现**：繁荣纪元的日志和废土上的NPC对话会动态反应玩家的行为，形成独一无二的个人史诗。
