## 1. 三种族系统（NPC 生成的根本需求）

### 1.1 种族定义

| 种族 | 起源 | 生物学特征 | NPC 形态生成需求 |
|------|------|-----------|------------------|
| **废土客** | 战后幸存者自然繁衍 20 代 | 真实人类，黏菌环境自然选择 | 当前 `npc.rs` 的 `HumanoidSkeleton` 已覆盖，需扩展年龄/体型/种族变异范围 |
| **虫族** | 银河系另一端的古老文明，世代飞船抵达 | 基因重组能力，每个个体独一无二；甲壳、触角、羽毛、喙等任意组合 | **必须支持任意拓扑骨骼**——不能假设人形 15 骨骼结构 |
| **克隆人** | 伊甸重启计划克隆体，被低语者篡改 | 基因优化的人类，可能携带变异片段 | 人形骨架 + 可选附加部位（额外肢体/器官） |

### 1.2 虫族生成关键约束

- **基因 Token 拼接**：身体由可拼接的功能 Token 组成，决定性状、代谢路径、外观
- **质体囊**：虫族细胞器，可捕获环境 DNA 整合到自身基因组
- **母巢重组繁殖**：巢母拆解基因片段重新组合，每个后代独一无二
- **形态多样性**：可能长甲壳+触角，也可能长羽毛+喙——同一文化认同但形态差异极大
- **程序化生成要求**：`Skeleton` 必须支持任意骨骼数量、任意层级深度、任意分支拓扑

### 1.3 母巢子实体（8 种异形生物，必须全部实现）

| 子实体 | 原型 | 形态 | 生命值 | 速度 | 数量 |
|--------|------|------|--------|------|------|
| **践踏者** | 模仿联军步兵 | 致密菌丝体，无眼无口，传感触角 | 低 | 中 | 极高 |
| **追猎者** | 模仿裂地兽四足掠食者 | 菌丝体+角质板，鞭状传感触角 | 中 | 极高 | 中 |
| **碎脊者** | 模仿动力装甲 | 厚重生物角质层天然装甲，1.5 倍人型 | 极高 | 低 | 极少 |
| **锈骑士** | 旧联军动力装甲+菌丝填充 | 装甲壳+菌丝血管脉动 | 极高 | 中 | 1-2/母巢 |
| **蜂群** | 模仿无人机 | 菌丝机翼+昆虫复眼+爆炸孢子囊 | 极低 | 中 | 高 |
| **臃肿者** | 巨型蜘蛛形态 | 膨胀腹部充满消化液 | 高 | 固定 | 中 |
| **窃听者** | 扁平地衣状 | 附着墙壁/岩石，被动感知 | 极低 | 固定 | 中 |
| **编织者** | 巨型蚂蚁形态 | 多条菌丝触手 | 中 | 低 | 低 |

**生成器要求**：每种子实体有独立 `MorphTemplate`，定义骨骼拓扑、几何体类型、材质参数、动画轨道。

## 2. NPC 程序化生成扩展需求

### 2.1 当前实现（Session 18）

`wasteland_render/src/procedural/npc.rs`：
- `HumanoidSkeleton::create`：15 根骨骼层级（pelvis→spine→chest→{neck→head, shoulders→arms}, thighs→calfs）
- `NpcBodyGenerator::generate`：程序化几何体（头球+颈圆柱+胸腰髋盒子+四肢圆柱）+ 蒙皮权重
- `create_walk_animation`：8 轨道步行动画
- `create_idle_animation`：呼吸动画

**局限**：仅支持人形，无法生成虫族、母巢子实体、变异生物。

### 2.2 扩展目标

#### A. 形态模板系统 `MorphTemplate`

```rust
pub struct MorphTemplate {
    pub name: &'static str,            // "humanoid" / "stalker" / "crusher" ...
    pub body_plan: BodyPlan,            // Bipedal / Quadrupedal / Insectoid / Amorphous
    pub skeleton_topology: Topology,    // 任意拓扑骨骼定义
    pub body_parts: Vec<BodyPartSpec>,  // 各部位几何参数
    pub material: BiologicalMaterial,   // 菌丝/角质/甲壳/血肉
    pub animations: Vec<AnimationTrack>,
}
```

#### B. 任意拓扑骨骼

`skeleton.rs` 已用 `SlotMap<BoneId, Bone>` + `parent: Option<BoneId>` 支持任意拓扑，但 `npc.rs` 硬编码人形 15 骨骼。需要：

- 新增 `SkeletonBuilder` 流式 API：`builder.add_bone(name, parent, local_bind)`
- 新增预定义拓扑常量：`HumanoidTopology` / `QuadrupedTopology` / `InsectoidTopology` / `AmorphousTopology`
- 母巢子实体各自一个拓扑常量

#### C. 多样化几何体

当前 `NpcBodyGenerator` 仅生成球+圆柱+盒子。需要扩展：

- **甲壳板**：扁平曲面（用 `sweep_along_path` 沿弧线扫描）
- **触角/鞭状结构**：渐变半径圆柱（沿骨骼链锥化）
- **菌丝血管**：贝塞尔曲线管道（新增 `bezier_pipe` 几何）
- **膨胀腹部**：椭球体（新增 `ellipsoid` 几何）
- **昆虫复眼**：半球阵列（多个小半球分布在大半球表面）
- **菌丝机翼**：薄膜结构（双面扁平网格 + 边缘加厚）

#### D. 变异系统

虫族基因 Token 重组产生的变异：
- **肢体数量变异**：0-6 条腿，0-4 条手臂
- **体型变异**：0.5x-3x 标准尺寸
- **附加器官**：额外眼、额外口、毒腺、发光器官
- **表面纹理变异**：菌丝密度、角质覆盖率、黏液分泌量

## 3. 建筑语义识别系统

### 3.1 当前实现

`wasteland_render/src/procedural/building.rs`：
- 参数化部件生成（窗框/人字屋顶/阳台/钢筋）
- `BuildingGenerator` 整合生成墙体+楼板+窗户+屋顶+阳台+钢筋
- **局限**：仅生成几何体，无类型标签，无功能分区，无承重分析

### 3.2 用户指出的问题

> "你这个建筑你又识别不了它是什么"

建筑必须能被引擎识别为具体类型（住宅/工厂/仓库/堡垒），且各功能区有明确语义。

### 3.3 扩展目标

#### A. 建筑类型分类（来自 v7.2 第三卷废土环境类型学）

| 类型 | 子类 | 关键特征 |
|------|------|---------|
| **居住区** | 民居/公寓/避难所宿舍 | 多窗户、小开间、生活设施痕迹 |
| **工业区** | 厂房/车间/仓库 | 大跨度、少内柱、高架起重设备 |
| **军事区** | 堡垒/哨所/弹药库 | 厚墙、射击孔、掩体、防爆门 |
| **商业区** | 商店/市场/办公楼 | 临街大窗、招牌位、开放空间 |
| **公共设施** | 学校/医院/电站 | 特殊功能房间（教室/手术室/发电机组） |
| **避难所** | 标准/种子/战争/实验/文明 | 见 §6 避难所分类 |

#### B. 建筑语义数据结构

```rust
pub struct BuildingSemantics {
    pub building_type: BuildingType,
    pub function_zones: Vec<FunctionZone>,
    pub structural_elements: Vec<StructuralElement>,
    pub load_bearing: LoadBearingGraph,
    pub age_and_decay: DecayState,
}

pub struct FunctionZone {
    pub zone_type: ZoneType,             // LivingRoom / Workshop / Storage / GuardPost ...
    pub bounds: AABB,
    pub access_points: Vec<Doorway>,
    pub fixtures: Vec<FixtureSpec>,      // 预装家具/设备
}

pub struct StructuralElement {
    pub element_type: StructuralType,    // LoadBearingWall / Column / Beam / Slab
    pub material: MaterialId,
    pub stress_capacity: f32,            // Pa
    pub current_stress: f32,             // 实时载荷
    pub damage: DamageLevel,
}

pub struct LoadBearingGraph {
    pub nodes: Vec<StructuralElementId>,
    pub edges: Vec<(StructuralElementId, StructuralElementId)>,
    pub root: StructuralElementId,       // 地基
}
```

#### C. 承重分析

- 关键承重部件毁坏 → 物理引擎计算连锁坍塌
- 应力传播：BFS 计算每根承重构件的深度和分组
- 坍塌阈值：超材料应力容量即断裂

#### D. 程序化生成集成

`BuildingGenerator::generate` 返回 `BuildingSemantics` 而非仅 `MeshBuilder`，让上层引擎能识别"这是什么建筑"+"哪些是关键结构"。

## 4. 角色损伤系统（用户重点要求）

### 4.1 用户原话

> "人物是会断腿 以及各种细节的 甚至你能看到血流动 包括骨骼等等各种的规范 都是需要真实化的改造"

### 4.2 四级损伤系统（来自 v7.2 第六卷）

| 等级 | 描述 | 视觉表现 | 功能影响 |
|------|------|---------|---------|
| **表面损伤** | 皮肤擦伤、轻度烧伤 | 纹理变化、轻度出血 | 无明显影响 |
| **中度损伤** | 切割伤、深层烧伤、肌肉撕裂 | 明显流血、组织外翻 | 该部位功能下降 30% |
| **重度损伤** | 骨折、肌腱断裂、器官损伤 | 大量出血、骨骼外露 | 该部位功能丧失 70% |
| **完全毁坏** | 肢体断裂、器官破碎、致命伤 | 残肢、骨碎片 | 该部位功能完全丧失 |

### 4.3 生理地图（Physiological Map）

身体分区域跟踪损伤，每个区域独立状态：

```rust
pub struct BodyRegion {
    pub region_id: BodyRegionId,         // Head / Torso / LeftArm / RightLeg ...
    pub skin_integrity: f32,             // 0..1，1=完好
    pub muscle_integrity: f32,
    pub bone_integrity: f32,
    pub vascular_integrity: f32,         // 血管完整性，影响出血速率
    pub nerve_integrity: f32,            // 神经，影响动作控制
    pub organs: Vec<OrganState>,         // 该区域内的器官
    pub foreign_bodies: Vec<ForeignBody>,// 弹片、箭矢等
}

pub struct OrganState {
    pub organ_type: OrganType,           // Heart / Lung / Liver / Stomach / Brain ...
    pub integrity: f32,
    pub bleeding_rate: f32,              // mL/s
    pub infection_level: f32,
}
```

### 4.4 血液流动模拟

> "甚至你能看到血流动"

- **血管网络**：跟随骨骼层级的简化血管树
- **血压系统**：心脏泵血 → 动脉 → 毛细血管 → 静脉回流
- **失血计算**：损伤部位血管完整性↓ → 出血速率↑ → 总血量↓ → 血压↓ → 休克/死亡
- **视觉表现**：
  - 表面伤口：纹理层血流（Shader 实现）
  - 中度伤口：粒子系统滴血
  - 重度伤口：粒子系统喷血（动脉破裂时喷射节奏与心跳同步）
  - 地面血泊：累积式纹理贴花

### 4.5 骨骼破坏

- **骨折类型**：闭合性骨折（皮肤未破）/ 开放性骨折（骨外露）/ 粉碎性骨折（多块碎片）
- **断裂判定**：冲击力 > 骨骼应力容量 → 断裂
- **断裂后果**：
  - 该肢体失去支撑能力（影响动作合成——见 §5）
  - 远端肢体失去神经控制
  - 内出血（骨髓血管破裂）
- **愈合**：长期固定 + 营养 → 骨痂形成 → 愈合（但可能畸形愈合）

### 4.6 跨系统耦合（v7.2 第六卷交互仲裁器）

```
物理碰撞检测 → 化学反应（酸/毒/火）→ 生物损伤评估
              ↓
         交互仲裁器
              ↓
   合并输出世界状态变更包
```

- 钝击：物理力 → 骨折 + 内出血
- 切割：物理力 + 锐度 → 皮肤裂开 + 外出血
- 穿刺：物理力 → 深度伤口 + 器官损伤 + 异物残留
- 烧伤：热能 → 皮肤碳化 + 神经坏死
- 酸蚀：化学反应 → 组织溶解 + 持续损伤
- 毒素：化学→血液→器官衰竭

## 5. 动作合成引擎（v7.2 第六卷 + 新建文本文档(4)）

### 5.1 核心思想

> "动作不是"播放"的，是"求解"的"

把动作拆成「意图」和「约束」，由系统实时计算出最终姿势。

### 5.2 架构

#### A. 原子动作库（50-80 个片段）

每个片段 0.3-1 秒，定义一种纯粹的关节运动：
- 躯干：直立 / 前倾 15° / 前倾 30° / 侧倾 / 扭转
- 右臂：前伸 / 上举 / 下探 / 侧捞 / 抓握
- 左臂：同上
- 腿部：站姿 / 走步循环 / 跑步循环 / 蹲姿 / 单膝跪
- 头部：正视 / 下看 / 侧看

#### B. 分层混合器

骨骼分为独立控制区，可同时播放不同原子片段：
- 下半身：走路/跑步循环（IK 适应地形）
- 躯干：根据目标位置动态前倾/侧倾/扭转
- 右臂/左臂：独立控制（捡东西、攻击、操作设备）
- 头部：注视目标

#### C. 意图解析器

```rust
pub struct ActionIntent {
    pub action_type: ActionType,         // PickUp / Attack / Operate / Throw ...
    pub target_entity: Option<EntityId>,
    pub target_position: Option<Vec3>,
    pub current_motion: MotionState,     // Standing / Walking / Running
    pub urgency: f32,                    // 0..1
}
```

意图解析器根据意图+身体状态+环境参数决定：
- 用哪只手
- 躯干弯曲多少
- 腿部继续跑还是短暂调整

#### D. 物理辅助

- 手部轨迹：弹簧-阻尼系统拉向目标，碰撞时弹开
- 抓取：手指姿态由被握物体形状实时决定
- 重量反馈：重物→手臂拉力↑→躯干下沉→步伐变重
- 过渡动作：惯性混合 0.2s 过渡

#### E. 与损伤系统集成（关键耦合）

损伤直接修改动作：
- 断腿 → 该腿无法承重 → 强制使用 SingleLegStanding / Crawling 拓扑
- 断臂 → 该手臂动作轨道禁用
- 神经损伤 → 动作抖动（添加噪声）
- 休克 → 整体动作迟缓 + 模糊

### 5.3 性能保障

- 原子动作库常驻内存 < 5MB
- 混合器计算 < 0.1ms/角色
- IK 仅对玩家和附近 NPC（LOD0）开启
- 物理辅助仅在接触时计算
- LOD 分级：远处角色用简化参数化动画

## 11. 用户"想不出来的东西"清单（主动发现）

基于文档扫描，以下需求用户未明确点名但文档已要求：

1. **损伤对动作的实时影响**（断腿不能站立 → 强制爬行）
2. **血液在地面的累积纹理**（不只是粒子，要有持久血泊）
3. **骨骼碎片作为可拾取物品**（断骨可作武器/工具）
4. **菌丝血管脉动动画**（锈骑士特有，与心跳同步）
5. **母巢子实体的菌丝触手物理摆动**（不是骨骼动画，是物理仿真）
6. **窃听者的扁平附着判定**（贴墙生成，需要表面检测）
7. **蜂群编队飞行**（群体行为，非个体动画）
8. **臃肿者死亡爆炸溅射**（死亡触发液体粒子系统）
9. **建筑承重图实时更新**（每根柱子被破坏后图重算）
10. **避难所类型决定内部布局**（不是统一模板，5 类各有布局规则）
11. **写实纹理的老化时间积分**（不只是静态纹理，是随时间演化的磨损）
12. **跨时代材料数据库**（燧石→超导陶瓷统一参数表）
