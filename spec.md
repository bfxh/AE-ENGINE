# wasteland_project 技术规格书

> **版本**: v4.2 — 属性驱动开放世界工厂模拟器
> **日期**: 2026-06-07
> **核心决策**: 不为架构纯洁性牺牲实用性。混用一切有效方法，在低配置设备上还原真实，让玩家创造出设计者未曾想到的东西。

---

## 0. 核心设计理念

### 0.1 规则取代配方
不存在"石斧=石片+木棍"的固定配方。所有物品功能由几何结构、材料属性和连接方式实时推导。

### 0.2 万物交互统一
物理、化学、生物、地质、气象不区分为独立系统。所有实体统一为"元体"，携带完整属性向量。统一响应函数同时输出力、属性变化和可能产物。

### 0.3 属性驱动一切
不是"铁+酸=反应"，而是"密度7.8+硬度HV200+含碳0.8%+体心立方晶格"遇到"pH=1的H+溶液"→系统根据属性向量推断反应。属性是世界的唯一真相来源。

### 0.4 规则泛化（Rule Generalization）
当玩家混合两种未知物质时，系统不查表，而是调用轻量级推理模型（纯Rust实现，微秒级延迟），根据双方属性在属性空间中的相似性推断结果。模型由离线化学引擎（RDKit）生成的海量虚拟实验数据训练，运行时只做推理，不做训练。

### 0.5 智能降频（Intelligent Frequency Scaling）
计算按紧迫程度分配——物理碰撞60Hz，铁剑氧化1秒1次，土壤肥力变化每分钟1次。99%的CPU留给玩家眼前的激烈战斗，远处石头的热传导不消耗任何资源。

### 0.6 玩家创造一切
玩家可以定义新元素（公理引擎）、设计新合金（材料学）、合成新化合物（化学）、培育新物种（生物）、建造工厂（自动化）。系统不预设任何配方，所有创造都是属性推理的自然结果。

### 0.7 公理引擎（Axiom Engine）
玩家可以声明新元素的物理属性，系统检查自洽性后生效。公理冲突时系统标记给社区裁决。实验验证依赖仪器，仪器本身也是公理——任何公理都可被质疑，科学永远开放。

## 一、架构总览

### 1.1 核心循环

```
┌─────────────────────────────────────────────────┐
│                  属性向量                          │
│  物理量 + 化学量 + 生物量 + 地质量 + 气象量         │
│  材料微观量 + 电磁量 + 声学量 + 光学量              │
└────────┬────────────────────────┬────────────────┘
         │                        │
    ┌────▼────┐              ┌────▼────┐
    │ 规则泛化器 │              │ 公理引擎  │
    │ (推理模型) │              │ (玩家定义) │
    └────┬────┘              └────┬────┘
         │                        │
    ┌────▼────────────────────────▼────┐
    │         统一响应函数                │
    │  力 + 属性变化 + 新元体生成          │
    └────┬────────────────────────────┘
         │
    ┌────▼────┐
    │ 智能降频  │
    │ 60Hz→0.1Hz│
    └─────────┘
```

### 1.2 混合方案原则
不追求单一架构。连续介质用MPM，离散结构用织网约束网络，场用有限差分，热力学用傅里叶方程，流体用Navier-Stokes。每种物理现象用最合适的求解器，不求统一。

### 1.3 动态边界元体
元体存在，但边界是动态的——岩石裂开时自动分裂为两个元体，金属熔化时两个元体自动融合。边界由应力梯度、化学浓度梯度和温度梯度实时决定。

## 三、规则泛化器

### 3.1 设计目标
不查表，不预设配方。当两个元体交互时，规则泛化器根据它们的属性向量推断交互结果。

### 3.2 实现方案
- **离线训练**：RDKit + 量子化学计算生成百万级虚拟实验数据
- **运行时推理**：纯Rust实现，使用微型决策树或浅层神经网络
- **延迟**：微秒级（<10μs），不阻塞主线程
- **输入**：两个元体的属性向量（各约100维）
- **输出**：交互类型 + 力向量 + 属性变化量 + 产物列表

### 3.3 泛化逻辑
```
已知: 铁(pH=1酸) → 溶解 + H₂↑
未知: 镍(pH=1酸) → ?
泛化: 镍与铁属性相似度0.87 → 推断镍也会溶解，但速率较慢
```

属性相似度由欧氏距离在归一化属性空间中计算。相似度越高，泛化置信度越高。

### 3.4 三级缓存
| 层级 | 匹配方式 | 延迟 |
|------|---------|------|
| L1 | 精确匹配（属性向量哈希） | 0.1μs |
| L2 | 模糊匹配（属性空间邻域） | 1μs |
| L3 | 规则泛化器推理 | 10μs |

## 四、公理引擎

### 4.1 核心机制
玩家可以声明新元素的物理属性，系统检查自洽性后生效。

### 4.2 自洽性检查
- 电负性与原子半径是否自洽
- 密度与原子量是否自洽
- 熔点与键能是否自洽
- 化学反应活性与电负性是否自洽

### 4.3 公理冲突
玩家A定义虚素在高温下分解，玩家B定义虚素在高温下聚合。两者各自内部自洽，但不能共存。系统标记冲突，提交社区裁决。裁决前，使用虚素的玩家会遇到不确定结果。

### 4.4 实验验证
- 仪器本身也是公理驱动的——测量依赖仪器的公理
- 玩家可以质疑仪器的公理，用另一种仪器验证
- 无限倒退由"公认测量标准"终止（社区默认信任某些仪器）
- 任何公理都可被质疑，科学永远开放

### 4.5 公理分叉
不同势力信奉不同公理集。战斗结果取决于"谁的公理集在当前环境下更强"，而非单纯战术。玩家倾向选择强势公理集，导致公理多样性减少——这是设计意图，不是Bug。

## 六、物理引擎（混合方案）

### 6.1 求解器分配

| 物理现象 | 求解器 | 说明 |
|---------|--------|------|
| 刚体碰撞 | 织网约束网络（排斥约束） | 无传统碰撞事件 |
| 连续介质（流体/沙/雪） | MPM物质点法 | 保留现有实现 |
| 建筑/机械结构 | 织网约束网络（固定+弹性约束） | 约束网络求解 |
| 断裂/破坏 | XPBD合规性矩阵平滑削减 | 非硬移除 |
| 热传导 | 傅里叶热方程 | GPU Compute Shader |
| 热对流 | 牛顿冷却定律 | 结合Navier-Stokes |
| 热辐射 | 斯特藩-玻尔兹曼定律 | 结合光谱属性 |
| 流体 | Navier-Stokes方程 | GPU加速 |
| 静电场 | 泊松方程 | 有限差分 |
| 静磁场 | 安培定律 | 有限差分 |
| 声波 | 波动方程 | 有限差分 |
| 光学 | 光谱路径追踪 | GPU Compute Shader |

### 6.2 动态边界元体
- 分裂条件：应力>极限强度 且 裂纹在约束图上传播到边界
- 融合条件：两元体温度>熔点 且 距离<0.001 且 化学相容
- 边界由应力梯度、化学浓度梯度、温度梯度实时决定
- 分裂/融合后自动更新约束网络

### 6.3 摩擦
不是预设系数。粗糙表面 = 更多约束节点 = 更多能量耗散 = 更大摩擦力。微观结构场影响表面粗糙度。

### 6.4 质量
- 内禀质量：元体的固有属性（`mass`字段），不随约束变化
- 有效惯性：内禀质量 + 相邻约束贡献的惯性项，各向异性

## 七、材料科学

### 7.1 微观结构场
每个元体内部有：
- **晶格方向场**：晶格主方向，决定各向异性
- **位错密度场**：位错分布，决定塑性变形能力
- **相分布场**：不同相（铁素体/奥氏体/马氏体）的空间分布

### 7.2 制造工艺影响属性
| 工艺 | 影响 |
|------|------|
| 淬火 | 改变相分布（奥氏体→马氏体），增加硬度，降低韧性 |
| 回火 | 马氏体→回火马氏体，降低硬度，恢复韧性 |
| 退火 | 消除位错，降低硬度，增加延展性 |
| 冷加工 | 增加位错密度，增加硬度，降低延展性 |
| 渗碳 | 表面含碳量增加，表面硬度增加 |
| 锻造 | 细化晶粒，增加韧性 |
| 时效 | 析出相形成，增加硬度 |

### 7.3 材料属性推导
```
含碳量0.8% + 马氏体相 + 晶粒直径10μm + 位错密度10^12/cm²
  → 硬度HV 800 + 断裂韧性 15MPa·√m + 弹性模量 210GPa
```

### 7.4 疲劳与蠕变
- 疲劳损伤累积：每次应力循环，`fatigue_damage`增加
- 疲劳极限：应力低于某阈值不累积损伤
- 蠕变：高温下持续应力导致`creep_strain`缓慢增加
- 腐蚀：暴露在腐蚀介质中，`corrosion_depth`按速率增加

## 八、地球系统

### 8.1 地质系统

#### 8.1.1 侵蚀
| 侵蚀类型 | 机制 | 速率 |
|---------|------|------|
| 雨水侵蚀 | 降水冲刷土壤 | 0.01mm/年 |
| 风蚀 | 风搬运细颗粒 | 0.001mm/年 |
| 冻融风化 | 水在裂缝中冻结膨胀 | 0.1mm/次 |
| 化学风化 | 酸雨溶解岩石 | 0.001mm/年 |
| 河流侵蚀 | 水流切割河床 | 1mm/年 |

#### 8.1.2 沉积
- 河流携带泥沙在下游沉积，形成三角洲
- 风搬运沙粒形成沙丘
- 湖泊/海洋底部沉积形成沉积岩

#### 8.1.3 板块构造
- 百万年尺度，仅在无人区加速
- 板块碰撞→造山运动
- 板块分离→裂谷
- 俯冲带→火山+地震

#### 8.1.4 矿山
- 矿物分布由地质历史决定
- 矿山可枯竭——玩家需要勘探新矿脉
- 矿石品位随开采深度变化

### 8.2 气象系统

#### 8.2.1 简化气象模型
- 温度、湿度、气压、风向、云量、降水
- 基于Navier-Stokes的简化大气模型
- 地形影响（山雨效应、焚风效应）

#### 8.2.2 极端天气
| 天气 | 条件 | 效果 |
|------|------|------|
| 暴雨 | 高湿度+低压 | 侵蚀加速、能见度降低 |
| 暴风雪 | 低温+高湿度 | 积雪、移动速度降低 |
| 沙尘暴 | 干旱+强风 | 能见度极低、腐蚀加速 |
| 龙卷风 | 强对流 | 破坏建筑、搬运物体 |
| 酸雨 | 污染+降水 | 加速腐蚀、伤害生物 |

### 8.3 水文系统
- 降雨→地表径流→河流→湖泊/海洋
- 地下水渗透和蓄水层
- 泉水、温泉（地热）
- 水坝可改变河流走向

### 8.4 生态系统

#### 8.4.1 动态生态
- 物种竞争/捕食/共生关系实时演化
- 种群数量动态平衡（Lotka-Volterra扩展）
- 入侵物种可能破坏生态平衡
- 环境变化触发适应性进化
- 玩家行为可能导致物种灭绝

#### 8.4.2 微生物生态
- 土壤菌群影响肥力
- 水体微生物影响水质
- 病原体传播和免疫
- 发酵和腐败

#### 8.4.3 基因漂变
- 小种群随机基因频率变化
- 长期隔离导致物种分化
- 玩家可培育新物种

## 九、声学交互系统

### 9.1 核心设计原则
声学不是"播放音效"，而是完整的物理模拟。声音在介质中传播受材质密度、弹性、温度、湿度、几何结构影响。声波不仅产生听觉，还产生物理力反馈，并作为Gameplay事件源。

### 9.2 SoundSource组件
所有可发声物体挂载SoundSource组件：

| 参数 | 类型 | 说明 |
|------|------|------|
| volume | f32 | 声源音量（dB SPL） |
| frequency_range | (f32, f32) | 频率范围（Hz） |
| directivity | DirectivityPattern | 指向性模式（全向/心形/8字形/枪形） |
| propagation_mode | PropagationMode | 传播模式（空气传导/固体传导/液体传导） |
| radiation_pattern | [f32; 36] | 36方向辐射强度（10°分辨率） |
| source_type | SourceType | 声源类型（点源/线源/面源） |

#### 9.2.1 声源类型
| 类型 | 体积衰减 | 示例 |
|------|---------|------|
| 点源 | 1/r² | 脚步声、枪声、NPC嗓音 |
| 线源 | 1/r | 高速公路、管道泄漏 |
| 面源 | 低频无衰减，高频1/r² | 大型机械、广播塔 |

#### 9.2.2 声源类别
| 类别 | 示例 | 优先级 |
|------|------|--------|
| 武器碰撞 | 金属碰撞、爆炸 | P0 |
| NPC嗓音 | 对话、呼喊 | P1 |
| 环境音景 | 风、雨、雷、黏菌蠕动 | P2 |
| 广播/播放器 | 便携音响、全息阵列 | P1 |
| 生物合成音 | 共振腔模型生成 | P2 |
| 建筑声学 | 结构振动、管道共鸣 | P3 |

### 9.3 声场求解器
在织网引擎中实现实时声场求解，或通过Go场服务器计算。

#### 9.3.1 波动方程
```
∂²p/∂t² = c² ∇²p
```
其中p为声压，c为介质声速。

#### 9.3.2 传播效应
| 效应 | 计算方式 | 影响参数 |
|------|---------|---------|
| 反射 | 镜像源法/射线追踪 | 材质密度、弹性、表面粗糙度 |
| 衍射 | UTD一致性衍射理论 | 障碍物几何、波长 |
| 吸收 | 频率相关衰减系数 | 材质吸收系数、温度、湿度 |
| 散射 | 表面粗糙度散射模型 | 表面不规则度、波长 |
| 折射 | Snell定律 | 介质密度梯度、温度梯度 |

#### 9.3.3 材质声学属性
| 属性 | 说明 | 影响 |
|------|------|------|
| acoustic_impedance | 声阻抗 | 反射/透射系数 |
| absorption_coefficient | 32波段吸收系数 | 频率相关衰减 |
| scattering_coefficient | 散射系数 | 漫反射程度 |
| surface_roughness | 表面粗糙度 | 高频散射 |
| porosity | 孔隙率 | 吸声能力 |

#### 9.3.4 环境声学参数
| 参数 | 说明 |
|------|------|
| 温度 | 声速变化（0.6m/s/°C），影响折射 |
| 湿度 | 高频吸收增强 |
| 风速 | 顺风/逆风影响传播距离和方向 |
| 气压 | 声速和衰减 |
| 几何结构 | 墙壁、废墟、地形影响反射和衍射 |

### 9.4 音频资产管理
所有音频资源按物理属性分类：

#### 9.4.1 材质碰撞声
| 材质对 | 生成方式 | 参数 |
|--------|---------|------|
| 金属-金属 | 物理建模（模态合成） | 密度、弹性模量、形状 |
| 金属-木材 | 混合模型 | 接触面硬度比 |
| 金属-石料 | 混合模型 | 接触面硬度比 |
| 木材-木材 | 物理建模 | 纤维方向、密度 |
| 石料-石料 | 粒子合成 | 碎片尺寸分布 |
| 玻璃-任意 | 断裂模型 | 裂纹传播速度 |

#### 9.4.2 环境音景
- 风：风速→湍流噪声频率和强度
- 雨：雨滴尺寸分布→冲击频谱
- 雷：放电能量→冲击波频谱
- 黏菌蠕动：生物力学→低频振动

#### 9.4.3 生物合成音
- 共振腔模型：根据生物体腔几何结构生成声音
- 声带物理模型：张力+气流→基频+谐波
- 牙齿/喙/爪碰撞：材质属性→碰撞频谱

### 9.5 音频空间化
Unity/Godot音频空间化插件通过声场数据实时驱动：

| 参数 | 来源 |
|------|------|
| 直达声延迟 | 声源距离/声速 |
| 早期反射 | 镜像源法最近N个反射面 |
| 后期混响 | 统计混响模型（RT60） |
| HRTF滤波 | 方向+头部几何 |
| 多普勒频移 | 相对速度 |
| 遮挡/衍射 | 射线追踪路径 |

### 9.6 物理力反馈
声波不仅产生听觉，还产生物理效应：

| 效应 | 触发条件 | 效果 |
|------|---------|------|
| 低频共振 | 频率匹配物体固有频率 | 手柄/物体振动 |
| 玻璃碎裂 | 声压>玻璃强度 | 特定频率触发裂纹 |
| 生物惊扰 | 声压>感知阈值 | 惊飞鸟类、惊跑动物 |
| 结构疲劳 | 持续高声压 | 微观裂纹扩展 |
| 滑坡触发 | 特定频率+地质条件 | 雪崩/山体滑坡 |

### 9.7 玩家体验流程

#### 9.7.1 声音播放
1. 玩家找到旧时代播放器或修复全息音响阵列
2. 选择音乐时，认知滤网提示"是否进入沉浸式聆听模式"
3. 音乐通过玩家自建的音响系统在环境中物理传播
4. 手柄随低频产生共振

#### 9.7.2 战斗应用
- 玩家可用播放器对准敌方设施发出特定音爆
- 定向高频脉冲进行基于物理的非致命干扰
- 利用环境声学（峡谷回声、建筑共振）放大效果

#### 9.7.3 声学工作坊
- 玩家可在基地建立声学工作坊
- 用不同材料（吸音棉、扩散板、隔音板）打造录音室
- 优化音质：测试不同材料的吸声系数和扩散特性

### 9.8 声音的记录与再现
- 玩家可使用旧时代录音设备或便携设备录制环境声音
- 声音通过设备扬声器在环境中物理传播，效果受空间材质和天气影响
- 录制的声音可存入物理介质进行交易
- 广播可吸引友方或敌对势力
- NPC会对录制的声音做出反应

### 9.9 光学

#### 9.9.1 光谱渲染
- 用32波段光谱替代RGB
- 每个元体有吸收光谱和发射光谱
- 光谱在介质中传播，不同波长吸收不同

#### 9.9.2 波动光学
- 衍射（光绕过障碍物边缘）
- 干涉（薄膜干涉、彩虹）
- 偏振（偏振片、应力双折射）
- 散射（瑞利散射=天空蓝色、米氏散射=云白色）

#### 9.9.3 次表面散射
- 光进入半透明介质（皮肤、玉石、蜡）
- 在介质内部散射后从另一位置射出
- 用扩散近似或路径追踪

## 十、信息传播

### 10.1 传播模型
- 信息有传播速度（信使、无线电、光信号）
- 信息会衰减（距离越远，细节越模糊）
- 信息会扭曲（传话游戏效应，每传递一次增加随机偏差）

### 10.2 知识网络
- 每个NPC有知识图谱（已知事实）
- 玩家行为产生"新闻事件"
- 新闻沿社会网络传播（旅行者、商人、信使）
- 不同势力有不同知识版本（同一事件的不同解读）

### 10.3 延迟效应
- A城事件→B城NPC知道：取决于A-B距离和传播速度
- 中世纪：步行/骑马速度（1-10天/100km）
- 工业时代：电报/无线电（即时但有范围限制）
- 废土：信使+无线电混合

## 十一、工厂系统

### 11.1 核心设计
玩家可以建造自动化工厂，将原材料转化为产品。工厂不是预设蓝图，而是由玩家组装的物理实体——传送带、熔炉、装配机、储罐、管道都由玩家设计。

### 11.2 工厂物理
- 传送带由织网约束网络驱动（旋转约束）
- 熔炉热力学实时计算（热传导+对流+辐射）
- 管道流体由Navier-Stokes求解
- 化学反应由规则泛化器推断

### 11.3 自动化
- 传感器检测物品属性（光谱分析、硬度测试）
- 执行器控制流程（阀门、开关、温度控制器）
- 逻辑控制器（玩家可编程：条件→动作）

### 11.4 产物质量
- 产物质量取决于原料属性+工艺参数+设备精度
- 温度波动→晶粒不均匀→硬度不一致
- 杂质含量→颜色/强度/导电性变化
- 没有"完美"产品，只有"在公差范围内"的产品

## 十七、玩家体验设计

> 绝对禁止"灵魂结晶"、"魔法"等任何非物理设定。所有体验都必须基于真实的物理交互。

### 18.1 装备的物理身份卡

**触发方式**：玩家在物品栏中右键装备，选择"检视"。镜头推近，装备缓慢旋转。

**感官呈现**：装备周围浮现废土科技风格的全息投影卡片，展示物理属性（重量、硬度、重心偏移）、可视化裂纹分布图和制造历史时间线。

**交互操作**：
- 旋转卡片、放大查看裂纹细节
- 切换简要/详细模式
- 选择"导出蓝图"

**反馈与后果**：检视后装备显示"已检视"标记。玩家通过身份卡了解装备的隐藏属性和疲劳临界点，辅助战前决策。

**叙事价值**：一把废土上流传的武器，其身份卡记录着历任使用者的历史。它不再是可替换的工具，而是承载着故事的伙伴。

### 18.2 环境感知的被动提示

**触发方式**：无需手动操作。角色进入特定环境时，认知滤网自动激活感官提示。

**感官呈现**：
| 环境 | 感官提示 |
|------|---------|
| 高湿度区 | 视野边缘有水雾感 |
| 酸性环境 | 呼吸声加重 |
| 高温区 | 视野有热浪扭曲 |
| 辐射区 | 视野出现噪点、滤网产生白噪声 |

**交互操作**：玩家可手动调整滤网敏感度，在信息丰富度与稳定性之间取得平衡。

**反馈与后果**：玩家从感官变化中自然推断环境信息，无需数值面板。长期暴露在恶劣环境中，装备腐蚀会加速，滤网会受损，玩家可根据提示提前防护。

**叙事价值**：在无数次穿越酸雾废墟后，玩家会形成本能的危险直觉。这是一种深层的沉浸感，来源于感官与物理世界的长期磨合。

### 18.3 装备的记忆碎片

**触发方式**：获得稀有装备时，身份卡上出现"加密数据片段"选项。使用中偶尔触发新碎片解锁提示。

**感官呈现**：激活时屏幕边缘短暂模糊，褪色、带噪点的全息影像叠加在装备上方，声音带有回声和失真。

**交互操作**：玩家可选择"激活"或"跳过"。已解锁的碎片可在身份卡中重播。

**反馈与后果**：每解锁一个碎片，玩家心理上与装备的联结加深。完整解锁后装备获"完全觉醒"标记，获得永久性小幅增益。

**叙事价值**：一把旧时代的狙击步枪，其碎片记录着前主人的最后战斗。当玩家身处类似绝境时，能感受到跨越时空的连接。

### 18.4 制造失败的意外发现

**触发方式**：拼装出功能置信度极低的组合体时，系统不报错，而是标记为"异常物品"存入背包。

**感官呈现**：异常物品在物品栏中有闪烁问号标记，身份卡显示"功能未知，建议送检"。挥动时，手柄传来不规则震动。

**交互操作**：玩家可将异常物品送往学院或企业进行物理测试鉴定，或自己进行反复实验。

**反馈与后果**：鉴定成功后，异常物品可能揭示隐藏的材料特性，甚至解锁隐藏的科技树节点。

**叙事价值**：废土上伟大的发现常源于意外。失败不再令人沮丧，因为每次失败都可能是一次新发现。

### 18.5 工厂的机械灵魂

**触发方式**：玩家靠近自动化工厂的控制终端，按下交互键。

**感官呈现**：屏幕上显示生产线实时数据（速度、温度、频率），故障节点闪烁红色并发出警报声。

**交互操作**：玩家可查看运行数据、调整参数、根据提示亲自前往故障点进行物理排查（如更换过热电机、磨损齿轮）。

**反馈与后果**：成功排除故障可获得"工程经验"加成。长期稳定运行后可解锁详细统计面板，获得成就感。

**叙事价值**：工厂是需要维护、有自己运行规律的复杂系统。玩家在维护中会产生类似照顾伙伴的情感连接。

### 18.6 废土的考古学

**触发方式**：在废墟深处发现"数据核心"。

**感官呈现**：修复后插入读取器，设备发出嗡鸣，投射出褪色、不稳定、伴有信号中断的全息影像。

**交互操作**：反复播放、逐帧分析、与其他碎片交叉比对以寻找隐藏关联。

**反馈与后果**：数据核心可能揭示隐藏地点、失传蓝图或纯粹的历史叙事。

**叙事价值**：一段来自繁荣纪元的家庭录像，虽无实用价值，却能让玩家感受到那个逝去世界的温度。

### 18.7 势力的记忆网络

**触发方式**：与NPC互动后，在个人日志查看"关系图谱"。

**感官呈现**：以玩家为中心的节点网络图，连线粗细代表关系强度，颜色代表关系性质，光晕代表NPC影响力。

**交互操作**：点击节点查看互动记录、当前态度和传闻。

**反馈与后果**：玩家的名声会通过商队等媒介传播到未探索区域，提前影响NPC的态度和交易。

**叙事价值**：玩家的每个选择都在社会网络中留下痕迹，感受到社会关系的真实与沉重。

### 18.8 时间的重量

**触发方式**：无主动触发，随游戏时间流逝自然呈现。

**感官呈现**：角色的恢复速度、镜中的面容、NPC的成长都在数十小时的跨度中缓慢变化。

**交互操作**：被动体验，无法干预。

**反馈与后果**：年长的角色经验更丰富，但体力和恢复速度会下降。

**叙事价值**：时间流逝的痕迹让玩家珍惜每一次冒险和相遇。

### 18.9 声音的记录与再现

**触发方式**：找到旧时代录音设备，或使用自己的便携设备。

**感官呈现**：声音通过设备扬声器在环境中物理传播，效果受空间材质和天气影响。

**交互操作**：玩家可录音、添加标签、编辑降噪，并将音频文件存入物理介质进行交易。播放时，认知滤网会提示"是否开启沉浸式聆听模式"。

**反馈与后果**：录制的声音可能被NPC听到并做出反应，广播可吸引友方或敌对势力。

**叙事价值**：声音是废土上最脆弱的遗物，一段久远的录音能给独行的玩家带来巨大的情感慰藉。

### 18.10 玩家自身的记忆传承

**触发方式**：当玩家结束此角色生命时（永久死亡或主动隐退），系统提示"是否将核心记忆编码为数据核心"。

**感官呈现**：编码时，玩家生命中重要瞬间以极快速度闪回，伴随当时情绪和身体感受。最终，所有记忆压缩为一块包含物理数据的数据核心。

**交互操作**：新角色在探索中可发现前世的"数据核心"，读取后会短暂体验前世的喜怒哀乐，并可能获得关键信息或技能倾向影响。

**反馈与后果**：数据核心可被其他玩家读取，每一次读取都会在核心上留下微弱的"回声"记录。

**叙事价值**：玩家不是在游玩孤立无关联的角色，而是在体验一条跨越生命的连续叙事。

## 十九、物理程序化动作系统

### 19.1 核心原理
角色没有预设动画文件。所有动作由织网引擎的约束网络实时求解。身体被抽象为一组由骨骼、肌肉约束连接的元体。当大脑发出动作意图时，引擎激活相应肌肉约束，驱动肢体运动。武器通过手持Socket与手臂形成固定约束，其质量、重心和速度实时反馈给整个手臂约束链。最终姿态是身体与武器形成的复杂约束网络在物理上唯一正确的解。

### 19.2 动作风格图谱
角色拥有一套动作风格图谱，由数十个关键原子姿势构成。每个姿势代表理想状态下某个动作的峰值形态。运行时，合成引擎根据角色当前速度、负重、武器重心、受伤部位等物理状态，在原子姿势之间进行物理混合与IK修正，由织网引擎的肌肉约束模型进行物理驱动微调，最终求解出独一无二的动作姿态。武器熟练度、基因改造、外骨骼装备等玩家状态作为数值参数输入混合空间，直接影响运动风格。

### 19.3 典型交互示例
#### 19.3.1 奔跑中捡东西
- 玩家在奔跑中按下交互键瞄准地面上的石头。角色上半身自然前倾，右手精准伸向石头，腿部继续奔跑但步幅略微调整以配合上半身动作。
- 石头很重时捡起瞬间身体明显下沉，手臂肌肉绷紧。在斜坡上时脚踝和膝盖自动弯曲适应地形。
- 手柄传来石头重量的反馈——重石传来沉闷的低频震动，轻石传来短促的轻微震动。
- 玩家只需按住交互键并持续奔跑，系统自动处理弯腰、伸手、抓取、恢复奔跑的全过程。
- 石头太重超出负重上限时角色尝试捡起但失败，身体后仰差点摔倒，玩家可放弃或选择减速停下来捡。
- 捡起的石头直接进入背包或手持，负重增加导致奔跑速度略微下降。反复捡重物导致肌肉疲劳累积，后续动作精度下降。
- 捡取成功与否完全取决于物理约束，不会出现手穿模或石头悬浮。

#### 19.3.2 战斗中武器脱手
- 玩家手臂受到敌人重击或手臂受伤导致握力不足。手臂被击中瞬间手柄传来剧烈的冲击震动。
- 手部约束断裂，武器从手中滑脱，在空中旋转飞出，落地时与地面碰撞产生真实的火花和弹跳。
- 手臂因突然失去负重而惯性上扬，身体重心失衡，视野短暂晃动，角色本能地后退一步恢复平衡。
- 武器脱手后玩家可选择立刻空手格斗、翻滚捡回武器、或切换备用武器。
- 捡武器需要实际走到武器落点弯腰拾取，不是在菜单中重新装备。
- 武器脱手是真实的物理结果，不是随机触发的惩罚机制。
- 手臂肌肉疲劳累积过高时握力阈值下降更容易脱手。装备手臂外骨骼时握力增强脱手概率降低。
- 武器脱手后可能被敌人捡走使用，NPC遵循同样的物理规则。

#### 19.3.3 动态攀爬与跑酷
- 面对障碍物时玩家按住攀爬键并移动摇杆，角色自主判断哪里可以攀爬、如何翻越。不依赖特定的攀爬点。
- 角色身体作为物理刚体与岩壁发生碰撞产生摩擦力，在狭窄缝隙中穿行时肩膀和臀部自动与岩壁接触。
- 大脑感知到速度变慢会加大向前移动的意图，脊椎在受到挤压时自动触发侧身的平衡反射。
- 每一次钻过缝隙的姿态都不同，不是预设动画。

### 19.4 性能优化
- 20米外的NPC只计算重心移动和碰撞，手臂和腿部简化为摆动的刚体，不运行完整IK。
- 静止的角色体内除维持呼吸的核心约束外所有肌肉约束休眠，计算成本接近零。
- 感知、决策、物理求解在不同帧上轮流执行，CPU每帧负载保持恒定。

## 二十、神经调控系统

### 20.1 核心原理
玩家通过认知滤网与自主神经中枢建立双向通信，可主动干预心跳、呼吸、代谢、部分免疫反应等自主功能。这不是魔法，而是真实的生理控制。认知滤网将玩家意图转化为精确的神经约束信号，调整相关器官元体的活动。调控有明确的物理代价：所有主动神经干预消耗远超正常状态的能量，高度依赖认知滤网稳定性。滤网受损或被干扰时被压抑的生理功能立即反弹，可能导致心脏骤停或呼吸衰竭。

### 20.2 潜行时的主动神经调控
- 玩家在潜行中打开认知滤网的神经调控面板，面板以3D全息投影形式悬浮在视野边缘，显示当前心率、呼吸频率、体温、代谢率等实时生理数据。
- 每个参数旁有可调节的滑块，滑块有明确物理上限。
- 调整滑块时角色身体产生对应生理反应——呼吸变慢时胸腔起伏幅度减小，心率降低时视野边缘脉搏跳动感减弱。
- 玩家拖动心率滑块降至极低水平，呼吸频率同步降低，体温调定点下调使皮肤表面温度逐渐与环境温度趋近。
- 完成设置后关闭面板，所有调控在后台运行。需要恢复时重新打开面板恢复正常值或设置定时自动恢复。
- 调控生效后玩家被生命探测仪发现的概率大幅降低，但身体进入低代谢状态导致移动速度和反应速度略微下降。
- 调控时间过长会加剧能量消耗需要补充食物。突然需要战斗时需要时间恢复心率，直接从低心率切换到高心率会导致心脏负荷骤增可能暂时晕眩。
- 这不是游戏给的潜行Buff，而是玩家用意志力主动控制了自己的身体。

### 20.3 神经结构编辑器
- 高阶玩法。玩家可在实验室或医疗舱中打开神经结构编辑器，看到自己或实验体的神经系统全息投影。
- 神经系统每个组件都是可编辑模块——感知神经、运动神经、中枢处理节点、自主调控中枢。
- 每种神经有参数：传导速度、信号衰减率、突触延迟、疲劳恢复速率。
- 玩家可拖动神经模块将它们连接在一起创建新的神经通路。
- 可将听觉神经与运动神经直接连接，创建听到特定频率就自动跳跃的反射弧，这不是脚本而是真实神经连接，可被后续训练强化或弱化。
- 脊柱受损导致下肢瘫痪时可用微电流桥接器直接连接大脑运动皮层和腿部肌肉，绕过脊髓损伤区域重建运动功能。
- 所有神经结构设计必须满足物理约束：材料需求包括特定蛋白质和离子通道蛋白，能量需求以ATP消耗速率为准，空间限制需考虑颅内空间有限。
- 超限时系统提示空间不足需要移除现有神经结构或扩大颅腔。
- 神经蓝图可保存、分享、交易，与其他蓝图使用相同格式。

### 20.4 虫族神经可塑性
- 虫族拥有最自由的神经可塑性，神经干细胞可分化任意类型神经元。
- 但代价极其昂贵——需要消耗大量生物质和能量。
- 虫族玩家在母巢中培育新神经结构移植体内，过程缓慢且有风险。

## 二十一、身体地图与沉浸式键位配置

### 21.1 核心原理
UI不再是屏幕上固定的界面，而是认知滤网根据玩家当前身体结构实时生成的动态图谱。玩家身体悬浮在面前，每一个改造组件、每一个原生器官都被标记。这是玩家与身体交互的唯一界面。

### 21.2 键位配置触发方式
- 玩家将鼠标向左移动，或在设备状态栏上选择键位配置。
- 也可以在任何时候通过特定按键或语音命令唤起。

### 21.3 键位配置感官呈现
- 玩家3D身躯悬浮在面前，背景就是当前所处的废土环境——营地篝火余烬在身后发光，废墟墙壁裂痕透过半透明身体隐约可见。
- 每个功能组件从体内发出微弱光芒——液压锤发出冷冽蓝光，触手发出暗绿色生物荧光，聚变反应堆透出橙红色脉动。
- 这些光不是UI特效，是设备本身待机时的物理发光。
- 屏幕底部有极简文字提示。
- 身体地图左侧悬浮两个设备图标对应键盘和手柄，视线扫过即可切换显示对应设备绑定。

### 21.4 键位配置交互操作
- 玩家将视线对准左臂液压锤，它感知到注视，蓝光变亮表明已激活。
- 按右键，全息小窗口从液压锤旁展开——不是矩形菜单，而是液压锤物理结构分解图，标注着握持方式、触发力度、反馈强度等可调参数。
- 玩家选择重击，用键盘按下空格键——从此空格键触发液压锤重击。
- 按左键，液压锤执行一次重击动作让玩家立刻看到效果。
- 想新增控制方式，在右键菜单选新增绑定，分解图上多出空白绑定槽，用手柄按下右扳机完成绑定。
- 两个控制方式并列显示互不冲突。
- 想删除某个绑定，右键绑定槽选择删除即可。
- 如果某个功能区域无任何绑定，不会在身体地图上显示。
- 每次绑定或调整后身体地图上的组件发出短暂闪烁确认。
- 如果绑定导致神经信号冲突，角色会表现出真实的不协调，认知滤网检测到异常并提示神经信号冲突询问是否需要调整。
- 这是真实的神经信号路由错误，不是Bug。如果玩家保留异常状态，身体会表现出异常行为，这是玩家对自己神经系统的真实改造结果。

### 21.5 恢复默认
- 键位配置界面右下角有一个几乎不可见的恢复默认标记——像一颗褪色的螺丝钉钉在废土地面上。
- 视线停留时它微微发光，按左键即恢复所有控制配置至出厂默认。
- 每个组件也有独立的恢复默认选项。
- 默认不是惩罚，是一个安全锚点，让玩家在无数次改造和实验中永远有可回溯的起点。

### 21.6 键位蓝图
- 所有键位配置可保存为蓝图，分享、备份、迁移。
- 他人身体结构相似可直接加载；结构不同则自动标记兼容与冲突部分。

## 二十二、沉浸式画质与图形配置

### 22.1 核心原理
画质调整不是跳出世界的菜单。认知滤网直接与视觉皮层双向通信，所有画面参数不是抽象滑块，而是视觉神经的真实物理调节。每个可调视觉元素都存在于废土环境中，它们就是环境本身。

### 22.2 所见即所得画质调整
- 玩家将鼠标移到任何视觉元素上——废墟墙壁的阴影、地面的材质、空气中的光柱、篝火的火星。悬停半秒激活，被悬停的视觉元素边缘微微发光。滚动滚轮直接调整对应参数，右键弹出详细参数窗。

#### 22.2.1 调整阴影
- 鼠标移到废墟墙壁阴影上，悬停激活。滚动滚轮调整阴影深度，所有阴影实时变深变浅。右键弹出参数窗可调阴影分辨率、阴影距离、软阴影质量，所有物体阴影同步实时变化。

#### 22.2.2 调整材质
- 鼠标移到脚下碎石地面，悬停激活。滚动滚轮调整材质分辨率，实时模糊变清晰。右键弹出参数窗可调材质过滤质量、各向异性过滤级别、纹理流送距离。

#### 22.2.3 调整全局光照
- 鼠标移到空气中孢子尘光柱上，悬停激活。滚动滚轮调整间接光照强度，周围环境实时变亮变暗。右键弹出更多GI参数。

#### 22.2.4 调整粒子密度
- 鼠标移到篝火火星上，悬停激活。滚动滚轮增减粒子数量。右键弹出粒子质量、粒子距离等参数。

### 22.3 约束与反馈
- 每个调整都有即时环境反馈。
- 视觉参数有生理上限——亮度不能超过视网膜损伤阈值，对比度不能超过视觉皮层饱和极限。
- 超限时滤网警告此设置可能损伤视觉神经。
- 追求极致画质需要付出视觉疲劳、暂时性眩光甚至永久性视觉损伤的代价。
- 视觉神经参数可保存为画质蓝图分享、备份、迁移。

### 22.4 设备全局配置
- 屏幕下方始终有一个极简设备状态栏，显示当前活跃输入设备。
- 鼠标移到图标上悬停激活，右键弹出设备全局配置：鼠标灵敏度、手柄死区、振动强度、按键重复延迟。
- 拖动振动强度滑块时手柄实时发出对应强度的振动。
- 所有调整都在当前环境中完成，不跳出任何界面。

## 二十三、神经编辑与义体改造的地点依赖性

### 23.1 核心原则

地点决定能力，技术决定上限。废土上不存在万能改造舱。不同的改造类型——生物神经、机械义体、数据重组——需要完全不同的设备、环境和专业知识。玩家必须在废土上寻找或建造对应的设施，才能进行特定类型的改造。同时，玩家自身的技术水平决定了能否绕过设施限制、自己动手改造——但这只是把麻烦从"找地方"转移到"自己慢慢搞"。

---

### 23.2 设施分级

#### 23.2.1 基础工作台

几乎任何安全屋或营地都可以搭建的基础设备。由简易金属框架、基础电源和手持工具组成，可能包含一个改造过的军用急救包或便携式诊断仪。工作台外观是拼凑的金属桌，工具挂在侧面，一盏旧时代的手术灯提供照明。

**可进行的操作：**
- 机械义体的简单维护和更换，如更换磨损的液压管、紧固松动的螺栓
- 外骨骼的穿戴调整和绑带更换
- 基础生化药剂的注射，如止痛剂、抗生素
- 简单神经信号校准，仅限已完全适配的义体
- 查看身体地图和键位绑定，进行软件层面的按键映射调整

**不可进行的操作：**
- 任何需要开腔的手术
- 神经系统的新增或重建
- 生物组织的培育和融合
- 数据重组和基因编辑

**玩家体验：** 玩家站在工作台前，身体地图自动展开，可进行已解锁的基础维护操作。每次维护消耗时间和基础材料，如润滑油、替换零件。维护不当会导致义体性能下降或故障。

---

#### 23.2.2 生物培育舱

需要在废土上找到或建造的中级设施。通常位于废弃的医疗机构、学院研究站或虫族母巢中。包含有机玻璃培养罐、营养液循环系统、生物质供应管线。空气中弥漫着营养液的甜腥味和微弱的腐殖质气息。培养罐中可能有正在生长的生物组织——触手、甲壳片、甚至完整的肢体。

**可进行的操作：**
- 生物组织的培育和移植，如肌肉增强纤维、生物甲壳质层
- 虫族的生物融合，将培育好的生物肢体与自身外骨骼融合
- 生物神经的修复和重建，如受损神经束的再生
- 简单的基因表达调控，如增强代谢率、促进伤口愈合
- 生物信息素腺体的培育和植入

**不可进行的操作：**
- 纯机械义体的精密制造
- 电子神经接口的植入
- 大规模数据重组
- 需要量子计算支持的神经结构设计

**虫族专属：** 虫族在母巢中拥有天然的生物培育舱，母巢的菌丝网络本身就是巨大的生物培育系统。虫族玩家可以在母巢中加速生物培育过程，消耗生物质来喂养培育中的组织。培育完成后，虫族通过痛苦而缓慢的生物融合过程将新组织整合入自身。

**玩家体验：** 玩家进入生物培育舱所在的房间，培养罐中的液体发出暗淡的绿色或蓝色荧光。玩家选择需要培育的生物组织类型，系统显示所需材料和培育时间。培育过程中，玩家可以透过培养罐的玻璃看到组织逐渐成形。培育完成后，玩家需要进行移植手术——如果自身技术水平足够，可在培育舱旁进行。如果不足，需要前往手术台或寻求NPC医生帮助。

---

#### 23.2.3 精密机械车间

需要在废土上找到或建造的中级设施。通常位于废弃的工厂、军团军工设施或企业的生产线上。包含数控机床、激光切割器、精密机械臂、电子元件焊接台。车间内充满金属切削的气味、电子焊接的焦味和冷却液的蒸汽。墙壁上挂着各种规格的螺栓、齿轮、轴承和电路板。

**可进行的操作：**
- 机械义体的精密制造和定制，如液压锤、精密抓取器、推进器
- 电子神经接口的植入和校准，如脑机接口、脊椎桥接器
- 外骨骼和动力装甲的深度改造，如增强承重、集成武器系统
- 义体武器的精密校准和升级
- 机械零件的批量生产和修复

**不可进行的操作：**
- 生物组织的培育和移植
- 生物神经的修复
- 基因编辑
- 需要生物培育环境的数据重组

**玩家体验：** 玩家进入精密机械车间，中央是巨大的数控机床和机械臂。墙壁上挂着各种规格的金属原料。玩家选择需要制造或改造的机械组件，系统显示所需材料和制造时间。制造过程中，机械臂在玩家面前精确加工零件。制造完成后，玩家需要进行安装——如果是简单的外骨骼，可直接在车间完成。如果是需要开腔的植入体，需要前往手术台。

---

#### 23.2.4 无菌手术室

需要在废土上找到或建造的高级设施。通常位于学院医疗翼、大型避难所或企业的高端诊所。完全密封的无菌环境，配备自动消毒系统、生命体征监测仪、精密手术机械臂、麻醉系统和血液循环维持设备。房间内充满消毒酒精和消毒剂的气味，灯光是冷冽的无影灯，所有器械都经过严格灭菌。

**可进行的操作：**
- 任何需要开腔的手术，如心脏替换、脊柱强化、颅腔扩展
- 神经系统的精密植入和重建，如脑机接口深度植入、脊髓桥接
- 生物和机械混合体的整合手术，如将生物触手与机械神经接口连接
- 高风险的数据重组和神经结构重编辑
- 大型改造项目，如全身义体化

**不可进行的操作：**
- 生物组织的长期培育（需要生物培育舱）
- 机械零件的精密制造（需要精密机械车间）
- 需要母巢环境的虫族生物融合

**玩家体验：** 玩家躺在手术台上，无影灯照在身上。如果玩家独自进行手术，需要通过认知滤网远程操控手术机械臂，这需要极高的专注度和神经接口稳定性。如果玩家技术不足，必须请NPC医生协助，可能需要支付费用、完成人情任务，或者承担手术风险。手术过程中，玩家可以看到自己的生命体征监测数据，以及手术机械臂在自己体内操作的实时画面。手术完成后，玩家需要恢复时间，期间不能剧烈活动或进行新改造。

---

#### 23.2.5 母巢生物质池（虫族专属）

虫族玩家在母巢中拥有的独特设施。不需要精密设备、无菌环境或量子计算机。母巢的菌丝网络本身就是巨大的生物培育和改造系统。

**可进行的操作：**
- 虫族所有类型的生物改造——神经节重组、肢体培育、甲壳融合、信息素腺体植入
- 通过消耗大量生物质，在生物质池中进行漫长的自我改造
- 不需要无菌环境或精密手术——虫族的身体改造是通过生物融合自然完成的

**不可进行的操作：**
- 精密机械义体的制造
- 电子设备的植入
- 需要量子计算支持的神经结构设计

**玩家体验：** 虫族玩家沉入母巢的生物质池中，温暖、黏稠的生物质液包裹全身。菌丝网络扫描虫族的整个身体，将改造需求转化为生物指令。虫族感受到自己身体内部的变化——新的肢体在体侧萌芽，旧的甲壳被新的替代，神经信号被重新路由。这个过程极其缓慢且痛苦，但完全自然，不需要任何外部设备。改造完成后，虫族从生物质池中爬出，身体焕然一新。

#### 23.2.6 量子神经编辑室（传说级设施）

废土上最稀有的设施。仅在繁荣纪元遗留的顶级研究设施、学院核心实验室或修复后的轨道站中存在。配备量子计算机、全息神经结构投影系统、单神经元精度编辑工具。

**可进行的操作：**
- 任何类型的改造——生物、机械、数据重组，均可在此完成
- 神经结构的从零设计——创造全新的感知模式、反射弧、多模态融合中枢
- 最高精度的义体制造和植入
- 大规模数据重组和基因编辑
- 认知滤网的深度升级和定制

**使用条件：**
- 极其稀有，废土上仅存在少数几个，且大多数被学院或未知势力控制
- 使用代价极高，可能需要完成漫长的任务链、支付天价资源、或与强大势力达成协议
- 操作复杂度极高，即使拥有设施，也需要极高的技术水平才能完全利用其功能

**玩家体验：** 进入量子神经编辑室如同进入圣殿——量子计算机以冷光闪烁，全息神经投影悬浮在空气中，每一个神经元都以亚微米精度呈现。玩家可以进行在普通设施中无法完成的终极改造——设计全新的感知器官、创造前所未有的神经通路、将自己的意识分布在多个身体中。但使用量子神经编辑室本身就是废土上最艰难的成就之一，获得使用权可能需要玩家投入数百小时的探索、战斗和外交努力。

---

### 23.3 基因重组引导与隔离

基因重组就像给运行中的电脑重装系统，不能带电操作。
- 学院或企业的高级实验室中有专门的基因编辑舱，提供完整的引导和隔离保障——全息投影展示完整基因图谱，标记可编辑Token位点和风险区域。
- 所有编辑操作在沙盒中先行模拟，验证通过后才真正写入。
- 编辑舱提供物理密封、独立供电、独立生命维持系统。
- 玩家也可以在营地中自己搭建简易隔离环境，用黏菌纤维布搭无菌帐篷，用蓄电池供电，用便携式紫外灯消毒。这种环境没有编辑舱的引导精度和生命维持备份，但玩家可以在里面做任何事。

---

### 23.4 熔断器

在进行任何基因编辑之前必须设计和安装熔断器。
- 熔断器是一段独立于被编辑基因区域的监控序列，持续监测关键生理指标——心跳、呼吸、神经信号传导、细胞凋亡率。
- 如果任何指标超出预设安全范围，熔断器自动触发强制回滚基因编辑操作。
- 熔断器设计本身是技术挑战——需要为每次基因编辑任务定制熔断器，设定合适的监控指标和回滚阈值。
- 阈值太宽松可能无法及时阻止基因崩溃，太严格可能频繁触发回滚导致编辑无法完成。
- 熔断器触发后被编辑的基因区域回滚到编辑前状态，玩家会感受到短暂不适但不会造成永久损伤。
- 熔断器本身也会消耗——每次触发都会烧毁部分序列，需要在下次编辑前重新合成。

---

### 23.5 基因崩坏

如果玩家在没有隔离的环境中进行基因编辑，或者没有安装熔断器就进行高风险操作，或者熔断器阈值设置不当导致未能及时触发，基因崩坏就会发生。
- 被编辑的基因Token发生不可控的连锁反应——原本只编辑肌肉增强基因，错误的剪切位点触发相邻代谢基因崩溃，然后连锁波及免疫基因、神经基因。
- 身体开始出现不可预测的变化，最终可能导致系统性崩溃，几乎无法逆转。
- 这不是系统给玩家的惩罚，而是玩家对基因编辑技术不尊重的物理代价。
- 玩家可以在死前留下数据核心，记录下自己的失败经验，供下一个角色或社区中其他玩家参考。

---

### 23.6 技术自主：当玩家自己掌握了改造技术

如果玩家通过学习和实践，自己掌握了神经编辑和义体改造的相关知识——这本身是一个漫长的技能累积过程——就可以在任何安全环境中进行改造，不需要依赖上述设施的完整功能。但这并不意味着改造变得容易。

**技术自主的代价：**
- 自己进行改造比使用专业设施慢数倍甚至数十倍
- 感染风险、操作失误风险、麻醉不足导致的疼痛和神经损伤风险都显著提高
- 自己操作无法达到专业设备的精度水平，部分高精度改造在设施外几乎不可能完成
- 某些改造——如需要量子计算机的神经结构设计、需要生物培养罐的组织生长——在设施外完全无法进行

**玩家体验：** 玩家在自己的营地中，打开便携式诊断仪，对着镜子进行手臂神经的重新校准。手指不稳定地握着微电流探针，每一次触碰都可能偏离目标神经元。认知滤网尽力提供引导，但精度远不如专业设备。整个过程持续数小时，期间玩家不能被打断——如果被敌人袭击，可能留下永久神经损伤。但当玩家终于完成校准，感受到手臂恢复控制时，那种成就感是使用专业设备无法比拟的。

---

### 23.7 玩家体验总结

废土上的身体改造不是菜单选项，而是需要探索、准备和执行的真实过程。你需要找到合适的设施，准备所需的材料，承担相应的风险，并接受改造后的恢复期。身体地图永远与你同在——它在任何设施中、任何环境下都能被唤起，帮助你理解自己当前的身体状态。你可以随时查看自己的改造历史、调整键位映射、进行基础的维护操作。但要进行真正的深度改造，你必须踏上寻找设施的废土之旅。

## 二十四、工厂系统（wasteland_factory）

### 24.1 核心原理

工厂不是生产菜单，而是由物理驱动的自动化流水线。传送带、熔炉、装配机、管道、能源网络——每个组件都是独立的物理实体，受真实物理规律约束。玩家设计布局、配置参数、维护设备，系统根据物理计算实时输出生产结果。

### 24.2 传送带系统（Conveyor）

#### 24.2.1 物理模型
传送带上的物品是独立物理实体，具有质量、摩擦系数、体积属性。传送带速度受电机功率、负载质量和磨损程度三重约束。

```
effective_speed = min(speed, max_speed) * (1.0 - wear * 0.5)
物品滑移 = 1.0 - friction * 0.1
功耗 = 总质量 * 有效速度 * 10.0
磨损率 = 物品数量 * 0.0001 * dt
```

#### 24.2.2 关键参数
| 参数 | 说明 |
|------|------|
| length | 传送带长度 |
| speed | 设定速度 |
| max_speed | 最大速度（电机限制） |
| wear | 磨损程度 (0.0-1.0) |
| power_consumption | 实时功耗 |
| throughput | 吞吐量统计 |

#### 24.2.3 故障模式
- 过载：物品总质量超过电机扭矩 → 传送带停转
- 磨损：长期运行 → 速度下降、功耗上升
- 卡料：物品摩擦系数异常 → 堵塞

### 24.3 熔炉系统（Furnace）

#### 24.3.1 热力学模型
熔炉基于斯特藩-玻尔兹曼定律和比热容计算精确温度变化。

```
热损失 = ε * σ * A * (T⁴ - T_ambient⁴) * (1 - insulation_quality)
加热功率 = input_power * efficiency * dt
温度响应 = (T_target - T_current) * heating_rate * dt * efficiency
净热量 = max(加热功率 + 温度响应 - 热损失*dt, 0)
ΔT = 净热量 / Σ(mass_i * specific_heat_i)
```

#### 24.3.2 熔化进度
材料在达到熔点时开始相变，吸收潜热期间温度不变，直到相变完成。

| 状态 | 条件 |
|------|------|
| 固态 | T < melting_point |
| 熔化中 | T == melting_point, progress < 1.0 |
| 液态 | T > melting_point, progress == 1.0 |

#### 24.3.3 关键参数
| 参数 | 说明 |
|------|------|
| temperature | 当前温度 |
| target_temperature | 目标温度 |
| power | 输入功率 |
| efficiency | 热效率 |
| insulation_quality | 保温质量 (0.0-1.0) |
| heat_loss | 实时热损失 |
| melt_progress | 熔化进度 (0.0-1.0) |

### 24.4 装配机系统（Assembler）

#### 24.4.1 精度模型
装配机基于配方定义将输入物品组合为输出物品。成品质量取决于输入物品精度和装配机自身精度。

```
成品质量 = 装配机精度 * 几何平均(各输入物品精度)
装配时间 = 基础时间 / (装配机精度 * 效率)
```

#### 24.4.2 配方系统
配方不是固定合成表，而是属性约束集合：
- 输入物品需满足的属性范围
- 输出物品的属性推导规则
- 装配精度要求

#### 24.4.3 关键参数
| 参数 | 说明 |
|------|------|
| precision | 装配精度 (0.0-1.0) |
| efficiency | 工作效率 |
| recipe | 当前配方 |
| input_slots | 输入槽位 |
| output_slot | 输出槽位 |
| progress | 装配进度 |

### 24.5 管道系统（Pipeline）

基于达西-魏斯巴赫方程计算流体传输。

```
压降 = f * (L/D) * (ρ * v² / 2)
其中 f = 摩擦因子, L = 管道长度, D = 管道直径
```

支持阀门控制（开度0.0-1.0）、流量调节、压力监控。

### 24.6 自动化控制系统（Automation）

#### 24.6.1 传感器与执行器
| 类型 | 功能 |
|------|------|
| 温度传感器 | 读取熔炉/管道温度 |
| 压力传感器 | 读取管道压力 |
| 流量传感器 | 读取传送带吞吐量 |
| 开关执行器 | 控制设备启停 |
| 阀门执行器 | 控制管道开度 |

#### 24.6.2 规则引擎
可编程的自动化规则：
- 条件触发：温度 > 阈值 → 关闭加热器
- 时序控制：每60秒 → 切换阀门
- 级联控制：上流停转 → 下流全部暂停

### 24.7 能源网络（Energy）

#### 24.7.1 能源类型
| 类型 | 来源 | 特点 |
|------|------|------|
| 电力 | 发电机、太阳能 | 通用能源 |
| 热能 | 熔炉、地热 | 直接热利用 |
| 机械能 | 风车、水车 | 直接机械驱动 |

#### 24.7.2 负载分配
能源网络自动计算各设备功耗需求，按优先级分配：
1. 安全关键设备（熔断器、冷却系统）
2. 生产设备（熔炉、装配机）
3. 辅助设备（传送带、照明）

#### 24.7.3 储能
电池组存储多余电能，效率衰减模型：
```
有效容量 = 标称容量 * (1.0 - 循环次数 * 0.0001)
```

## 二十五、公理引擎（wasteland_axiom）

### 25.1 核心原理

公理引擎允许玩家声明新元素、新材料、新物质的物理属性，系统检查自洽性后生效。这不是"作弊"——玩家定义的公理必须通过物理一致性检查、社区投票裁决和实验验证三道关卡才能被世界接受。

### 25.2 公理定义（Axiom）

#### 25.2.1 数据结构
```rust
Axiom {
    id, name, domain,           // 基础标识
    properties: HashMap<String, f32>,  // 属性字典
    relations: Vec<AxiomRelation>,     // 与其他公理的关系
    declared_by: String,               // 声明者
    version: u32,                      // 版本号
    status: AxiomStatus,               // 状态
}
```

#### 25.2.2 公理领域
| 领域 | 说明 |
|------|------|
| Material | 材料属性（密度、硬度、熔点等） |
| Element | 元素属性（原子序数、电负性等） |
| Compound | 化合物属性 |
| Biological | 生物属性 |
| Energy | 能量属性 |

### 25.3 自洽性检查（Consistency）

#### 25.3.1 内置检查规则
| 检查规则 | 属性对 | 期望关系 |
|----------|--------|----------|
| 电负性-原子半径 | electronegativity, atomic_radius | 负相关 |
| 密度-原子质量 | density, atomic_mass | 正相关 |
| 熔点-键能 | melting_point, bond_energy | 正相关 |
| 反应活性-电负性 | reactivity, electronegativity | 正相关 |
| 热膨胀-比热容 | thermal_expansion, specific_heat | 正相关 |
| 硬度-键能 | hardness, bond_energy | 正相关 |

#### 25.3.2 自洽性评分
```
consistency_score = max(0, 1.0 - errors*0.3 - warnings*0.1)
```
- score >= 0.7: 通过
- score < 0.7: 需要修正
- score = 0: 严重冲突，拒绝生效

### 25.4 冲突检测与裁决（Conflict）

#### 25.4.1 冲突类型
| 类型 | 说明 |
|------|------|
| DirectContradiction | 同一属性值直接矛盾 |
| IncompatibleImplications | 属性推导结果矛盾 |
| DomainOverlap | 领域覆盖冲突 |
| ExperimentalDisagreement | 实验验证结果不一致 |

#### 25.4.2 社区投票机制
检测到冲突后，系统发起社区投票。参与者（NPC/玩家）投票选择：
- SupportA: 支持公理A
- SupportB: 支持公理B
- Synthesize: 综合两者
- RejectBoth: 拒绝两者

#### 25.4.3 裁决规则
| 投票结果 | 条件 | 裁决 |
|----------|------|------|
| 支持A >= 67% | A获胜 | 公理A成为标准 |
| 支持B >= 67% | B获胜 | 公理B成为标准 |
| 综合 >= 67% | 综合方案 | 生成新公理 |
| 拒绝 >= 67% | 两者皆否 | 都标记为无效 |
| 支持A+综合 >= 67% | 分裂领域 | 按使用场景划分 |

### 25.5 公理分叉（Fork）

#### 25.5.1 分叉机制
不同势力可以维护自己的公理集。当势力对同一物理现象的理解不同时，公理分叉产生。

```
AxiomFork {
    parent_axiom_ids: 继承的公理集合
    axioms: 势力专属公理
    faction: 所属势力
    dominance: 支配度 (0.0-1.0)
    follower_count: 追随者数量
}
```

#### 25.5.2 环境适应性
公理分叉在不同环境条件下表现不同：
```
combat_effectiveness = dominance * Π(1/(1 + |env - optimal| * scale))
```
- 高温环境：适应性强的公理分叉表现更好
- 高压环境：不同分叉的压强适应性差异显现

### 25.6 实验验证（Experiment）

#### 25.6.1 仪器系统
仪器本身也是公理——依赖于底层公理的测量结果。任何公理都可被质疑。

| 仪器类型 | 测量属性 | 精度范围 |
|----------|----------|----------|
| Spectrometer | 光谱 | 0.8-1.0 |
| Thermometer | 温度 | 0.9-1.0 |
| HardnessTester | 硬度 | 0.7-1.0 |
| ChemicalAnalyzer | 化学成分 | 0.6-1.0 |

#### 25.6.2 测量信任链
```
测量值 = 真值 * 校准系数 + 噪声 * (1-精度) * 随机符号
置信度 = 精度 * 仪器信任度
```

#### 25.6.3 交叉验证
两个独立仪器测量同一属性，一致性越高，结果越可信：
```
一致性 = 1 / (1 + |测量值A - 测量值B|)
```

## 二十六、信息传播系统（wasteland_info）

### 26.1 核心原理

信息在废土上不是瞬时的。知识、谣言、新闻、信号都需要时间和介质传播，在传播过程中会衰减、失真、被放大或被压制。这不是"消息系统"——这是物理世界中的信息传播。

### 26.2 知识图谱（Knowledge）

#### 26.2.1 事实节点
```rust
KnowledgeNode {
    id, content,           // 基本信息
    confidence: f32,       // 置信度 (0.0-1.0)
    decay_rate: f32,       // 衰减率
    last_verified: f32,    // 最后验证时间
    source: String,        // 来源
    related_nodes: Vec<String>,  // 关联节点
    tags: Vec<String>,     // 标签
}
```

#### 26.2.2 知识衰减
未经验证的知识随时间衰减：
```
当前置信度 = 初始置信度 * e^(-decay_rate * elapsed_time)
```
衰减到阈值以下时，知识节点标记为"不可靠"。

#### 26.2.3 关联查询
知识图谱支持基于内容、标签和关联网络的多维度搜索。

### 26.3 信号传播（Propagation）

#### 26.3.1 传播模型
| 模型 | 适用场景 | 衰减公式 |
|------|----------|----------|
| Radio | 无线电通信 | 自由空间路径损耗 |
| Optical | 光信号/信号弹 | 比尔-朗伯定律 |
| Acoustic | 声音传播 | 球面波衰减 |
| Seismic | 地震波 | 介质衰减 |
| Thermal | 热信号 | 热扩散 |

#### 26.3.2 传播衰减
```
信号强度 = 发射功率 * e^(-衰减系数 * 距离) / 距离²
```
- 障碍物额外衰减：穿墙 *0.1, 穿山 *0.01
- 天气影响：雨雪额外衰减, 雾天光信号衰减

### 26.4 信息失真（Distortion）

#### 26.4.1 失真模型
| 失真类型 | 说明 | 触发条件 |
|----------|------|----------|
| Mutation | 信息内容随机改变 | 每次传播有小概率 |
| EmotionalAmplification | 情感内容被放大 | 高情感负载信息 |
| Simplification | 复杂信息被简化 | 多次传播后 |
| Bias | 信息被立场扭曲 | 派系传播 |

#### 26.4.2 谣言检测
```
谣言概率 = 1 - 置信度 * 来源可信度 * 交叉验证次数/总引用次数
```

### 26.5 社交网络（Network）

#### 26.5.1 社交代理
```rust
SocialAgent {
    id, faction,           // 基础信息
    knowledge: KnowledgeGraph,  // 个人知识图谱
    trust_network: HashMap<id, trust>,  // 信任网络
    influence: f32,        // 影响力
}
```

#### 26.5.2 新闻扩散
新闻事件通过社交渠道传播：
```
传播半径 += 10.0 * 重要性 * dt
派系认知度 += 信任度 * 0.1 * dt
```
- 新闻重要性越高，传播越快
- 信任度越高，接受越快
- 派系隔离：敌对派系信息传播受阻

#### 26.5.3 传播延迟
信息不是即时到达的。距离越远，延迟越大：

## 二十八、角色物理系统（wasteland_character）

> **实现**: `wasteland_character` crate | **测试**: 16 tests passing
> **依赖**: `wasteland_physics`, `wasteland_weave`

### 28.1 表面约束网络

角色碰撞不再使用胶囊体近似——而是由元体属性驱动的表面约束网络。碰撞的结果不是预设的"弹开"，而是材料在极限状态下的真实响应。

```
骨骼几何 → 表面统计 → 接触检测 → 织网约束 → 约束求解 → 塑性变形/断裂 → 力反馈
```

### 28.2 骨骼系统

关节类型: Fixed, Hinge(axis/limit), Ball(swing_limit), Universal(axis1/axis2), Slider(axis/limit)

身体部位: Head, Torso, UpperArmL/R, LowerArmL/R, HandL/R, UpperLegL/R, LowerLegL/R, FootL/R, Tail, WingL/R, Custom(u32)

每个骨骼节点包含: 局部位置/旋转、长度、半径、关节(刚度/阻尼/摩擦)、质量(自动计算)、致命标记

### 28.3 表面微观几何统计（surface_stats.rs）

表面不在代码里存储三角网格——存储的是统计特征参数：

```rust
SurfaceStats {
    roughness,           // 表面粗糙度 Ra [0, 1]
    texture_direction,   // 纹理方向（如木纹）
    bump_density,        // 微凸体密度
    bump_height_mean,    // 凸起高度均值
    bump_height_stddev,  // 凸起高度标准差
    hardness,            // 表面硬度
    friction_coefficient,// 摩擦系数
}
```

预定义材质：metal_blade(roughness=0.05), stone_rough(0.85), wood_grain(0.5), flesh(0.3), iron_plate(0.3)

**微观接触面生成**: 两个表面碰撞时，基于双方统计参数随机生成符合统计特征的微观接触面(MicroContactPatch)，包括微凸体数量、接触点位置/法线、总接触面积。

### 28.4 表面约束网络接触检测（surface_contact.rs）

替代胶囊体GJK检测，使用线段最近点逼近骨骼碰撞：

- 线段-线段最近点算法检测两个骨骼是否接触
- 基于穿透深度和骨骼半径估计接触面积
- 生成 SurfaceConstraintPair，包含两面材质的 SurfaceParams
- 表面约束: 屈服强度 = hardness_a * hardness_b * 2.5e7, 极限强度 = yield * 2.0

Weave 约束系统新增 **ConstraintType::Surface**:
- force < yield: 弹性变形，回到原形状
- yield < force < ultimate: 塑性变形（permanently modify rest_length）
- force > ultimate: 约束断裂（apply_damage）

**自碰撞过滤**: 忽略父子骨骼对，只检测非相邻骨骼。

### 28.5 塑性变形累积（deformation.rs）

每次碰撞在表面留下永久痕迹——不是贴花，是真实的形变累积：

- PlasticDeformation: 位置、深度、时间戳、力方向、材质
- DeformationHistory: 累计应变、磨损深度(cumulative*0.01)、锋锐度损失(depth*0.1)
- DeformationTracker: 每个骨骼独立追踪
- 磨损(sharpness_loss>0.3)使刃口变钝，粗糙度增加，硬度下降

### 28.6 力反馈投影（force_feedback.rs）

手柄传来的不是预设振动波形，而是约束求解过程中真实力变化的实时投影：

| 事件类型 | 触发条件 | 频率公式 |
|---------|---------|---------|
| SurfaceSlide | 摩擦滑动 | roughness*200 + speed*50 |
| BreakThrough | 约束断裂(力骤降) | 30Hz脉冲 |
| Stuck | 持续高阻力 | 10Hz低频 |
| MicroVibration | 粗糙表面滑动 | roughness*300 + speed*100 |
| ConstraintBreak | 超极限强度断裂 | 60Hz高频 |

ForceFeedbackBus 维护活跃事件列表，get_combined_force(time) 返回当前时刻的总反馈力向量。

### 28.7 织网约束系统扩展（wasteland_weave）

**SurfaceParams**（新增到 ConstraintEdge）:
```rust
SurfaceParams {
    yield_strength,       // 屈服强度
    ultimate_strength,    // 极限强度
    hardness,             // 硬度
    friction_coefficient, // 摩擦系数
    surface_roughness,    // 表面粗糙度
    contact_area,         // 接触面积

}
```

## 三十、存储系统（wasteland_storage）

> **实现**: `wasteland_storage` crate | **测试**: 24 tests passing
> **依赖**: `serde`, `serde_json`, `bincode`

### 30.1 快照系统（snapshot.rs）

存档格式基于自定义二进制容器，包含魔数校验、版本控制和CRC完整性检查。

```
WSTL 魔数 (4B) | 版本 (4B) | SchemaVersion | 帧号 | 时间戳 | 实体数 | 压缩大小 | 原始大小 | CRC32 | 压缩数据
```

**Snapshot 生命周期**:
1. `Snapshot::new(schema, frame, timestamp, raw_data)` → 自动RLE压缩 + CRC32计算
2. `snapshot.serialize()` → bincode序列化到磁盘
3. `Snapshot::deserialize(data)` → 反序列化 + 魔数/版本/CRC三重校验
4. `snapshot.decompress()` → 解压恢复原始数据
5. `snapshot.verify_integrity()` → 独立CRC校验

**SaveFile**: 管理多个快照和增量，支持追加写入和大小查询。

### 30.2 增量压缩（delta.rs）

基于实体ID的差分计算，避免全量存储：

```
WorldDelta {
    frame_from, frame_to,
    entities: [EntityDiff { entity_id, action: Created/Removed/Modified, component_data }],
    global_state: ...
}
```

- `compute_entity_diff(before, after)`: 逐实体比较二进制数据，输出新增/删除/修改
- `apply_delta(state, delta)`: 应用增量到当前状态

## 三十八、动画生成系统（wasteland_ai_tools::animation_gen）

> **实现**: `wasteland_ai_tools/src/animation_gen.rs` | **测试**: 4 tests passing
> **依赖**: `serde`, `rand`

### 38.1 核心原理

不使用预设动画文件。所有角色动画由物理约束网络实时求解，animation_gen模块提供程序化动画生成、逆运动学(IK)求解和动画混合能力，作为AI工具链的动画生成后端。

### 38.2 动画类型

```rust
pub enum AnimationType {
    Walk, Run, Jump, Climb, Attack, Idle, Death, Interact, Custom(String),
}
```

### 38.3 动画生成器（AnimationGenerator）

**AnimationRequest**: 动画类型 + 持续时间 + 速度因子 + 风格参数 + 骨骼配置

**AnimationResult**: 关键帧序列 + 关节轨迹 + 根运动增量 + 事件标记 + 总持续时间

**关键帧**: 时间戳 + 关节姿态数组 + 根位置/旋转 + 事件标记

**事件标记**: 脚步落地、攻击命中、跳跃离地/到顶点/落地

**生成流程**:
1. 根据动画类型选择基础运动模式
2. 应用速度因子和风格参数调整
3. 生成关键帧序列
4. 填充中间帧（线性插值）
5. 后处理（根运动平滑、IK修正）

### 38.4 动画混合

```
blend_animations(from, to, blend_time):
  for each joint:
    blended_rotation = slerp(from.rotation, to.rotation, t)
    blended_position = lerp(from.position, to.position, t)
  return blended_result
```

### 38.5 IK求解器（IkSolver）

**CCD（Cyclic Coordinate Descent）算法**:
- 从末端效应器向根节点迭代
- 每次迭代调整一个关节角度使末端逼近目标
- 参数: max_iterations(默认100), tolerance(默认0.001), damping(默认0.5)

**关节定义**: 位置 + 旋转 + 长度 + 父关节索引 + 旋转约束(轴+限制)

**关节姿态**: 旋转四元数 + 平移向量

**性能**: 单链CCD求解 ≤ 0.5ms（典型骨骼链）

### 38.6 运动合成（MotionSynthesis）

**MotionStyle**: 运动风格参数
- speed_multiplier: 速度倍率
- stride_length: 步幅
- body_lean: 身体前倾角
- arm_swing: 手臂摆动幅度
- crouch_factor: 蹲伏程度

**合成流程**:
1. 根据速度计算步态周期
2. 生成腿部关节正弦运动
3. 应用骨盆垂直振荡
4. 手臂对侧摆动
5. 脊柱旋转补偿
6. 头部稳定

### 39.1 核心原理

基于种子值的确定性世界生成。所有生成结果只依赖种子，相同种子产生完全相同的世界。支持地形、生物群系、水体、结构放置、资源散布和生成点生成。

### 39.2 生物群系

```rust

    Desert, Tundra, Grassland, Forest, Rainforest,
```

**群系分布规则**:

### 39.4 水体模拟

**水体类型**: Ocean(海洋), Lake(湖泊), River(河流), Wetland(湿地), Aquifer(地下含水层)

**河流生成**:
- 从高海拔源点沿梯度下降
- 汇合形成更大河流
- 入海口形成三角洲
- 流量 = 上游集水面积 * 降水量

### 39.5 结构放置

**StructureType**: House, Tower, Wall, Bridge, Road, Ruin, Bunker, Facility, Custom(String)

**放置规则**:
- 平坦地形优先（坡度 < 15°）
- 道路连接主要结构
- 废墟随机分布在远离聚落区域
- 设施按功能分布（矿场近矿脉、农场近水源）
- 避免重叠（碰撞检测）

### 39.6 资源散布

**ResourceType**: IronOre, CopperOre, Coal, Stone, Wood, Water, Oil, RareEarth, Crystal, Custom(String)

**矿脉生成**:
- 基于地质层的矿脉分布
- 富集度随深度增加
- 稀有矿物（稀土/水晶）出现在特定地质构造

**资源密度**: 由群系类型和地质层共同决定

### 39.7 生成点系统

**生成点类型**: PlayerStart, NpcSpawn, EnemySpawn, CreatureSpawn, VehicleSpawn, ItemSpawn, EventTrigger

**放置规则**:
- 玩家初始点：安全区域，远离敌对生物
- NPC生成点：聚落内或道路旁
- 敌对生物：远离初始区域，夜间密度增加
- 物品生成：遵循稀有度分布

### 39.8 地牢生成

**DungeonGenerator**:
- BSP（二叉空间分割）算法划分房间
- 走廊连接相邻房间
- 房间类型：普通/宝箱/Boss/陷阱/休息
- 种子确定性：相同种子产生相同地牢
