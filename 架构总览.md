## 0. 项目定位

### 0.1 一句话定义

**AE-ENGINE**是一款基于底层科学规则的物理沙盒生存制造游戏。玩家扮演从培养舱中苏醒的克隆体，在黏菌覆盖的废土上，用双手拼装一切——武器、工具、建筑、甚至生命本身。没有合成菜单，没有配方书，没有预设动画。所有玩法由物理、化学、生物规则实时推导产生。

### 0.2 八大核心哲学

| # | 哲学 | 含义 |
|---|------|------|
| 1 | 规则取代配方 | 系统不做法官，只做翻译官 |
| 2 | 失败即玩法 | 没有错误，只有意外发现 |
| 3 | 世界即工作台 | 环境是工具，制造是物理交互 |
| 4 | 质量永不妥协 | 规则不裁剪，优化靠算法革命 |
| 5 | 知识自由流动 | 开源协议默认，社区驱动知识库 |
| 6 | 涌现产生故事 | NPC 记忆、生态演化、社会博弈 |
| 7 | 万物交互统一 | 物理/化学/生物不再区分为独立系统 |
| 8 | AI 与游戏一体 | 游戏世界训练 AI，AI 反哺游戏 |

### 0.4 不可推翻的技术约束

- **Rust + winit 0.30 + wgpu 24** 三件套不变
- **glTF 为主要资产格式**，动画/蒙皮/骨骼均从 glTF 2.0 导入
- **Blender 为唯一外部编辑器**，引擎内部编辑器只做关卡/属性/脚本
- **GPU Compute 先于 CPU compute**，DSL/反应/折叠/菌丝全部 GPU compute
- **推荐配置 RTX 3080+**，不照顾低端 GPU
- **仅支持 PC**（Windows、Linux、SteamOS），不支持手机

## 第一部分 · 引擎层

### 1.1 引擎层定位

引擎层提供"科学规则运行时"，是物理/化学/生物/场/事件等底层子系统的集合。引擎层不定义任何玩法（玩家/按键/任务/HUD 均属游戏层）。

**核心原则**：
- 引擎只提供机制，不提供策略
- 所有实体统一为"元体"（MetaEntity），携带完整属性向量
- 物理/化学/生物不是独立引擎，而是同一元体上的不同响应函数

### 1.5 生化链 L0→L8（核心差异化设计）

> **核心原则**：所有生物实体必须经过 L0→L8 生化链验证才能存在于世界中。AI 走独立的 A0→A8 信息链。两条链可交叉但不混淆。

| 层级 | 名称 | 构建时 | 运行时 |
|------|------|--------|--------|
| L0 | 元素表（42 个） | 属性模板 | 查表 |
| L1 | 化学键规则（14 键型） | 价态/电负性/ΔG 判定 | 查表 |
| L2 | 官能团（18 核心） | 兼容性矩阵 18×18 | react(A,B,cond) |
| L3 | 小分子/单体库 | 氨基酸22/核苷酸6/单糖12/脂类8/辅因子10 | 属性查询 |
| L4 | 聚合反应规则 | 缩合/自由基/配位/自组装 | 链终止判定 |
| L5 | 折叠/构象规则 | Chou-Fasman + 疏水塌缩 + MC SA | 查表 + 后台精细 |
| L6 | 分子机器 | Michaelis-Menten 动力学 | 功能调用 |
| L7 | 结构组装 | 五层组装（膜/骨架/能量/菌丝管） | 使用预组装 |
| L8 | 实体快照 | EntityCard 属性卡 | tick 代谢 |

**运行策略**：链不是每秒跑一次。是**建造时跑一次，产物快照被引擎使用**。玩家创造新东西→链重跑→新快照替换旧的。

**渲染对接**：
- 菌丝网络可视化（GPU particle 沿体素网格）
- 感染者表面菌丝纹理叠加（decal / second UV）
- 感染者视野色调变化（post-process color grading）
- 折叠结果 3D 坐标可视化（spline mesh / 球棍模型）

### 1.6 GPU Compute Kernel 规格

**统一粒子缓冲区（GPU SoA）**：
```
position[P] | velocity[P] | stress[P] | material[P]
temperature[P] | substances[P][8] | concentrations[8]
folding_state[P] | entity_card_id[P]
```

**反应引擎 WGSL**：
- 每个粒子 8 个物质槽位 + 浓度
- Arrhenius 速率常数 `k = A·exp(-Ea/RT)`
- pH 修正因子
- 反应热 `delta_H` 反馈到粒子温度
- 链式反应最大深度 5 层

**折叠引擎 WGSL**：
- Phase 1（即时 <100ms）：Chou-Fasman + 疏水塌缩
- Phase 2（后台 <10s）：Monte Carlo 模拟退火，5 项能量（E_burial/E_contact/E_hbond/E_compact/E_ss）
- MC 移动操作：片段替换 60% / 局部扰动 25% / rotamer 切换 10% / 域刚体移动 5%
- 模拟退火：T_start=500 / T_end=0.5 / 100 cycles / 几何冷却 0.95
- 16 链并行（RTX 4090 上 N=500 ~4s）

**MPM 与反应耦合**：
```wgsl
@compute @workgroup_size(64)
fn mpm_p2g(@builtin(global_invocation_id) gid: vec3<u32>) {
    // ... 标准 MPM P2G ...
    let new_T = /* 从 MPM 计算出的新温度 */;
    if new_T > particle.reaction_threshold_T {
        particle.temperature = new_T;
        particle.needs_reaction_pass = 1u;  // 下一帧 reaction_pass 使用新温度
    }
}
```

## 第二部分 · 渲染层

### 2.1 渲染层定位

渲染层提供"视觉呈现运行时"，从引擎层读取数据，输出到屏幕。渲染层不定义任何玩法逻辑。

- **ae_render（v1）**：已备份到 `D:\AI\storage\CC\2_Old\`，准备开源
- **nova_render（v2）**：借鉴 bevy/rend3/kajiya/Fyrox 重构，维护性更好

### 2.2 nova_render 架构

借鉴四个开源引擎：
- **bevy**：双 World 分离（MainWorld↔RenderWorld）、四阶段（Extract/Prepare/Queue/Render）、PipelineCache、乒乓纹理
- **rend3**：Handle+Arc 资源管理、RenderGraph、Megabuffer、Material Trait ABI、profiling crate
- **kajiya**：后处理栈顺序、RenderGraph（handle+temporal+imageops）
- **Fyrox**：Pool+强类型Handle、Visit trait IR、Prefab 继承、UUID 资源、GraphicsServer trait

#### 2.5.3 四层建模架构（渲染对物化生的核心支持）

1. **材质层**：PBR 材质库，参数由元体化学/生物属性实时驱动

   - 微生物活跃度 → 菌斑
   - 温度 → 黑体发光
   - 辐射 → 色调偏移
2. **原子几何体层**：三角网格 + 材质标签 + 物理属性 + 几何特征标记 + Socket 点 + 碰撞体
3. **组合体层**：功能推导引擎扫描几何特征输出功能置信度
4. **动态表现层**：Shader 在 GPU 完成物理/化学/生物/环境驱动变化
   - 应力超过屈服强度 → 顶点位移
   - 断裂 → 沿裂纹分裂为子模型
   - 氧化进度 → 材质色混合与锈斑
   - 腐蚀深度 → 置换贴图
   - 基因 Token → 基础形态
   - 微生物代谢 → 外观变化
   - 全局温度/湿度/孢子浓度 → 所有 Shader 参数

### 4.3 游戏系统清单

#### 4.3.1 元体系统与交互溶解架构（核心）

> "游戏底层不存在独立的物理引擎、化学引擎和生物引擎。所有实体统一为元体，每个元体携带完整的属性向量。"

**元体属性向量**：
- 物理量：质量/密度/硬度/韧性/弹性模量/屈服强度/极限强度/泊松比/摩擦系数/弹性恢复系数/温度/导热率/比热容/电导率/磁导率
- 化学量：元素组成/化学键类型/反应活性/酸碱度/氧化还原电位/氧化进度/腐蚀深度/化学污染度/溶解度/可燃性/毒性
- 生物量：基因 Token 序列/代谢速率/生长速率/修复速率/神经信号强度/生命值/辐射剂量/毒素浓度/营养水平/水合作用/细胞类型/组织密度

**交互响应函数**（同一物理步长内按序计算，无先后顺序）：机械响应 → 化学响应 → 热响应 → 生物响应 → 电响应 → 氧化响应

#### 4.3.2 结构场与应力传播

- 应力传播沿预计算路径，每次传播衰减 30%
- 深度限制防止无限递归
- 关键节点破坏后结构崩溃连锁效应

#### 4.3.3 功能推导引擎与制造系统

> "规则取代配方：不查表，根据几何特征+材料属性+质量分布实时推导物品功能。"

**推导维度权重**：
- 切割：锐边(0.4) + 硬度(0.3) + 长宽比(0.03) + 韧性(0.2)
- 穿刺：尖端(0.4) + 硬度(0.3) + 细长比(0.2) + 质量集中度(0.1)
- 钝击：质量(0.5) + 硬度(0.3) + 接触面积反比(0.2)
- 容器：空腔体积(0.6) + 内壁平滑度(0.2) + 材料惰性(0.2)
- 支撑：平坦表面(0.5) + 强度(0.3) + 接触面积(0.2)

武器置信度 0~1，分四级：通用道具、钝器、简易武器、精良武器。**任何物体都可攻击。**

#### 4.3.4 化学玩法

- 血条：倾斜菱形/斜线，半透明，静止 5 秒淡出，移动/攻击 0.2 秒恢复
- 默认键位：WASD 移动 / C 蹲 / Space 跳 / Shift 冲刺 / Alt 或双击方向 闪避 / 1-8 快捷栏 / Tab 物品栏 / M 地图 / Esc 设置
- **联机时不暂停**：打开物品栏/制造/设置游戏世界继续运行，会被攻击

#### 4.3.14 环境设计体系

- **可读性**：3 秒内理解空间功能，视觉线索提升导航满意度 42%
- **流线**：主路径 70% 玩家（宽≥3m）/ 支路径 20% / 秘密路径 10%
- **节奏**：每 2-4 分钟切换张力区/喘息区
- **地标系统**：一级 3km 可见 / 二级 500m / 三级 50m
- **空间叙事密度**：高密度 8-15 元素/100m² / 中 3-7 / 低 1-2 / 零 0
- **线索引导梯度五级**：环境暗示→物件→光照→符号→直接引导
- **叙事节拍**：每 45-90 秒一次
- **物件定位三级**：一级 Hero Props 每 50m² 2-3 个 / 二级 Support 每 10m² 3-5 个 / 三级 Filler 每 1m² 5-10 个

#### 4.3.15 动作合成引擎

> "动作不是'播放'的，是'求解'的。"

- 原子动作库：约 50-80 个原子片段（每个 0.3-1 秒）
- 分层混合器：下半身/躯干/左右臂/头部独立控制
- 意图解析器：类型+目标+移动状态+紧急程度
- 物理辅助：弹簧-阻尼系统 / 抓取 IK / 重量反馈
- 性能：原子动作库常驻内存<5MB，混合器<0.1ms/角色

#### 4.3.16 自动化物流系统

- 初期即有：用木头石头搭建基础传送带、分拣器
- 真实模拟：传送带物品有物理碰撞，堆叠过高掉落，效率受零件磨损、天气影响
- 后期升级：电力驱动、无人机物流

#### 4.3.17 生物驯养系统

- 范围：不仅动物，真菌、植物（黏菌、食肉藤）也可驯化
- 机制：基于生物引擎，成功率受食物、环境、基因亲和度影响

#### 4.3.18 地图与雷达系统

- 地图非上帝视角，而是雷达装置扫描生成的实时俯视图
- 没有雷达就没有地图
- 迷雾：未探索区域被战争迷雾覆盖，探索后永久可见

#### 4.3.19 AI 导演系统

- 动态管理敌人刷新、战斗节奏（参考《星际战斧》）
- 单机和联机均适用

#### 4.3.20 打击感系统

- 材质碰撞粒子 / 物理布娃娃 / 屏幕震动 / 音效合成共同驱动（参考《黑神话》）
- 音效由材质对和力度实时合成，无预制文件

#### 4.3.21 游戏内游戏

- 废土中找到旧世界掌机/电脑，可玩复古 2D 小游戏
- 不搞 2D 视角主模式

### 4.4 游戏层对引擎/渲染/编辑器的接口需求

#### 4.4.1 对引擎层

- **物理**：确定性物理核心（Jolt 默认 + Rapier 备选，通过 PhysicsTrait 抽象层切换）；IPC 作为 GPU 加速模块处理非确定性高级效果（建筑坍塌碎片/大规模颗粒堆积/布料软体/流体飞溅）

- **生物**：BioModels 数据库 → COPASI 解析 SBML → 简化参数适配游戏时间尺度（三层时间：实时层查表 / 秒级层竞争模型 Lotka-Volterra / 分钟级层代谢网络）

- **交互仲裁器**：所有物化生事件通过统一仲裁器（优先级：急性物理 > 化学 > 生物，合并输出世界状态变更包）

- **AI 智能体**：GOAP 战略规划 → 效用 AI 战术决策 → 行为树执行 → 记忆图谱持久化
- **存档**：分层存储（热数据内存 ECS / 温数据异步写入磁盘 / 冷数据 SQLite）

#### 4.4.2 对渲染层

- **四层建模架构**（见 2.5.3）
- **角色/NPC 渲染**：IK 系统（手掌吸附武器握把、脚部贴合地形）/ 捏脸系统（MakeHuman 基础上大改，面部至少 30 个可调参数）
- **环境渲染**：体积光 / 尘埃散射 / 雾效密度（安全区>500m / 探索区 200-500m / 危险区<100m）/ 废土配色方案
- **UI 渲染**：所有 UI 元素均为 3D 空间实体（Sprite3D, Label3D, 自定义 Mesh），无传统 HUD / 半透明斜向菱形血条/小地图 / 全息投影/浮动符文属性面板

- L1/L2 编辑器（见第三部分）

## 第五部分 · 四层接口契约

### 5.1 接口方向

```
游戏层 ──调用──→ 引擎层（EngineContext trait）
游戏层 ──调用──→ 渲染层（通过 EngineBridge 提取数据，不直接调渲染私有 API）
游戏层 ──调用──→ 编辑器层（L3/L4 游戏内编辑器）
编辑器层 ──调用──→ 引擎层（公开 API，不访问私有模块）
编辑器层 ──调用──→ 渲染层（Viewport 渲染）
渲染层 ──读取──→ 引擎层（EngineBridge::extract() 每帧提取数据）
```

### 5.2 禁止的依赖方向

- ❌ 引擎层依赖游戏层
- ❌ 渲染层依赖游戏层
- ❌ 引擎层依赖渲染层
- ❌ 编辑器层访问引擎私有模块（必须通过公开 trait）

### 5.3 灰色地带处理原则

引擎提供机制，游戏层提供策略：
- 渲染参数（雾密度/光照强度）由游戏层通过引擎数据驱动
- GBuffer 组成由渲染层决定，但材质参数由游戏层填充
- 后处理链顺序由渲染层固定，但参数由游戏层调节
